`docs/` 目录现在只保留项目级、跨 Spec 的全局文档。

- `docs/家庭版OpenClaw-详细设计方案-v0.1.md`
- `docs/家庭版OpenClaw-系统架构图说明与数据库设计草案-v0.1.md`

- `docs/家庭版OpenClaw-产品Backlog与里程碑甘特-v0.1.md`

## 放在 `specs/<spec-id>/docs/` 的文档

与某个 Spec 直接对应的交付文档，统一放到该 Spec 目录下，便于关联和归档。

### `Spec 001`

路径：`specs/001-家庭底座与成员中心/docs/`

当前包含：

- `specs/001-家庭底座与成员中心/docs/家庭版OpenClaw-首批接口文档-v0.1.md`
- `specs/001-家庭底座与成员中心/docs/家庭版OpenClaw-本地启动与联调说明-v0.1.md`
- `specs/001-家庭底座与成员中心/docs/家庭版OpenClaw-第一轮联调验收清单-v0.1.md`
- `specs/001-家庭底座与成员中心/docs/家庭版OpenClaw-第一轮联调验收结果-v0.1.md`

## 归档规则

- 全局规划、架构、跨阶段设计文档：放 `docs/`
- 某个 Spec 的接口、联调、验收、补充说明：放 `specs/<spec-id>/docs/`
