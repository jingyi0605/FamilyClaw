## 2026-03-17 当前边界

当前正式插件类型已经是 9 类：

- `connector`
- `memory-ingestor`
- `action`
- `agent-skill`
- `channel`
- `locale-pack`
- `region-provider`
- `theme-pack`
- `ai-provider`

其中 `theme-pack` 和 `ai-provider` 已经纳入通用插件系统，不再是主题系统和 AI 供应商系统各自维护的平行概念。

如果你现在要改插件启用/禁用规则，先看：

- `docs/开发设计规范/20260317-插件启用禁用统一规则.md`

如果你现在要改版本字段边界，先看：

- `specs/004.5-插件能力统一接入与版本治理/docs/20260317-插件版本治理现状与最小能力说明.md`

这份开发者手册负责讲“怎么开发、怎么声明、哪些边界别越线”，不再单独维护另一套主题插件规则或 AI 供应商插件规则。

如果你现在改的是下面两类东西，先看中文 `03` 和 `05`，再动代码：

- `region-provider` 节点返回结构
- `household_region_context` 的消费逻辑

因为从 `2026-03-18` 开始，家庭地区上下文里正式补了统一 `coordinate` 结果。上层插件应该直接读这个结构，不要再从 `city` 文本猜坐标。

这里同时维护中文和英文两套版本，要求如下：

- 中文：`zh-CN/`

- 两边目录结构和主题保持一一对应，不要一边有、一边没有

## 维护原则

插件开发文档以后按两层维护：

- 稳定规范：讲边界、规则、不要做什么，尽量少改
- 高频变化引用：讲 API、样例、当前文件位置，允许改，但只保留一份事实来源
