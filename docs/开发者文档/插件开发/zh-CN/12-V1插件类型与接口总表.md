# 12-V1插件类型与接口总表

这份文档不是讲理念的，是给你对着开发的。

## 1. 正式插件类型总表

| 类型 | 用途 | 当前口径 | 典型入口 |
| --- | --- | --- | --- |
| `integration` | 持续状态接入 | 正式执行型插件 | `entrypoints.integration` |
| `action` | 动作执行 | 正式执行型插件 | `entrypoints.action` |
| `agent-skill` | Agent 工具能力 | 正式执行型插件 | `entrypoints.agent_skill` |
| `channel` | 通讯通道 | 正式执行型插件 | `entrypoints.channel` |
| `region-provider` | 地区与坐标解析 | 正式执行型插件 | `entrypoints.region_provider` |
| `ai-provider` | AI 供应商插件 | 供应商声明 + `entrypoints.ai_provider` driver | `entrypoints.ai_provider` |
| `locale-pack` | 语言包 | 资源型插件 | 无 |
| `theme-pack` | 主题包 | 资源型插件 | 无 |

## 2. 记忆独占槽位总表

| 槽位 | 用途 | 入口 |
| --- | --- | --- |
| `memory_engine` | 事件到记忆候选的提取、合并、重要度评估 | `entrypoints.memory_engine` |
| `memory_provider` | 记忆索引、存储、检索 | `entrypoints.memory_provider` |
| `context_engine` | 上下文构建、摘要、压缩 | `entrypoints.context_engine` |

## 3. `integration` 现在真正开放了什么

状态型插件当前的正式主链路，不是“随便返回点设备数据”。

插件至少要理解这几个输出位：

- `instance_status`
- `devices`
- `entities`
- `discoveries`
- `actions`
- `diagnostics`
- `dashboard_snapshots`
- `records`
- `observations`

可选补充位：

- `message`
- `summary`
- `items`
- `sync_scope`

运行时上下文里，和天气、多实例同步最相关的入口是：

- `_system_context.region.household_context.coordinate`
- `_system_context.integration_runtime.database_url`
- `_system_context.integration_runtime.db_session`（仅内置插件同进程链路）

## 4. `ai-provider` 的补充说明

当前统一口径是：

- 宿主保留统一 AI 网关、路由、权限、审计、密钥、插件状态校验和降级
- `ai-provider` 插件负责供应商声明、字段 schema、driver entrypoint、协议适配、流式输出、厂商特例

如果你在旧 spec、旧报告或旧代码里看到旧说法，那是历史背景，不是当前规则。

## 5. 禁止事项

- 不要再声明 `connector`
- 不要再声明 `memory-ingestor`
- 不要把新的 AI 供应商或新的厂商特例继续写回核心
- 不要把历史兼容层误写成正式规范
