这里放给插件开发者直接使用的中文文档。

当前文档默认按“第三方插件最终走同容器子进程 runner”这条路线来组织。

要先看清一个现实边界：

- 当前仓库已经实现的是内置插件同进程运行底座
- 第三方插件的推荐目标模式改为“主服务 + 同容器子进程 runner + 插件自带 venv”
- 对外执行主路径已经切到“先创建后台任务，再由 worker 执行插件”
