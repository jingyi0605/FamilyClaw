# 06-从零开发一个可运行插件

## 文档元数据

- 文档目的：给开发者一条从 `manifest` 到接口调用的完整落地路径，照着做就能拼出一个当前架构下可运行的插件。
- 当前版本：v1.0
- 关联文档：`docs/开发者文档/插件开发/zh-CN/01-插件开发总览.md`、`docs/开发者文档/插件开发/zh-CN/02-插件开发环境与本地调试.md`、`docs/开发者文档/插件开发/zh-CN/03-manifest字段规范.md`、`docs/开发者文档/插件开发/zh-CN/04-插件目录结构规范.md`、`docs/开发者文档/插件开发/zh-CN/05-插件对接方式说明.md`
- 修改记录：
  - `2026-03-13`：创建首版 walkthrough，覆盖从 `manifest`、代码文件到接口调用的完整示例。

这篇不是规范解释，而是实战流程。

目标只有一个：

- 从零写出一个后续 runner 能执行、主项目也能识别的插件

## 1. 先选一个最简单的目标

不要一上来就做动作插件，也不要一上来就做高风险控制。

这篇 walkthrough 选的目标是：

- 做一个最小的健康数据同步插件
- 插件类型：`connector + memory-ingestor`
- 结果：通过一次同步接口，把步数和心率写进系统记忆

插件 id 用这个：

- `health-demo-sync`

## 2. 第一步：放到正确目录

第三方插件推荐不要直接塞进主项目源码树。

先建自己的插件目录：

```text
health-demo-sync/
  manifest.json
  requirements.txt
  README.md
  plugin/
    __init__.py
    connector.py
    ingestor.py
```

仓库目录名可以自由一点，但 `manifest.id` 还是用中划线。

## 3. 第二步：写 `manifest.json`

先放最小可运行版本：

```json
{
  "id": "health-demo-sync",
  "name": "健康演示同步插件",
  "version": "0.1.0",
  "types": ["connector", "memory-ingestor"],
  "permissions": [
    "health.read",
    "memory.write.observation"
  ],
  "risk_level": "low",
  "triggers": ["manual", "agent-checkpoint"],
  "entrypoints": {
    "connector": "plugin.connector.sync",
    "memory_ingestor": "plugin.ingestor.transform"
  },
  "description": "读取演示健康数据，并写成 Observation 记忆。",
  "vendor": {
    "name": "FamilyClaw Demo",
    "contact": "internal-demo"
  }
}
```

这里最容易犯的错：

- `id` 写成下划线
- `entrypoints` 模块路径和 `plugin/` 下真实目录对不上
- 声明了 `memory-ingestor`，但没配 `memory_ingestor` 入口

## 4. 第三步：写 `plugin/connector.py`

这个插件先返回两条原始记录：步数和心率。

```python
from __future__ import annotations


def sync(payload: dict | None = None) -> dict:
    normalized_payload = payload or {}
    member_id = normalized_payload.get("member_id", "demo-member")
    captured_at = normalized_payload.get("captured_at", "2026-03-13T07:30:00Z")

    return {
        "source": "health-demo-sync",
        "mode": "connector",
        "received_payload": normalized_payload,
        "records": [
            {
                "record_type": "steps",
                "external_member_id": member_id,
                "member_id": member_id,
                "value": 9032,
                "unit": "count",
                "captured_at": captured_at
            },
            {
                "record_type": "heart_rate",
                "external_member_id": member_id,
                "member_id": member_id,
                "value": 68,
                "unit": "bpm",
                "captured_at": captured_at
            }
        ]
    }
```

这个返回值满足当前系统最关心的东西：

- 有 `records`
- 每条 record 都有 `record_type`
- 有 `value`
- 有 `captured_at`

## 5. 第四步：写 `plugin/ingestor.py`

现在把原始记录转成 Observation。

```python
from __future__ import annotations


def transform(payload: dict | None = None) -> list[dict]:
    normalized_payload = payload or {}
    records = normalized_payload.get("records", [])
    observations: list[dict] = []

    for record in records:
        if not isinstance(record, dict):
            continue

        raw_id = record.get("id")
        payload_data = record.get("payload", {})
        if not isinstance(payload_data, dict):
            payload_data = {}

        record_type = record.get("record_type")
        subject_id = payload_data.get("member_id") or payload_data.get("external_member_id")
        observed_at = record.get("captured_at")

        if record_type == "steps":
            observations.append(
                {
                    "type": "Observation",
                    "subject_type": "Person",
                    "subject_id": subject_id,
                    "category": "daily_steps",
                    "value": payload_data.get("value"),
                    "unit": payload_data.get("unit", "count"),
                    "observed_at": observed_at,
                    "source_plugin_id": "health-demo-sync",
                    "source_record_ref": raw_id,
                }
            )
        elif record_type == "heart_rate":
            observations.append(
                {
                    "type": "Observation",
                    "subject_type": "Person",
                    "subject_id": subject_id,
                    "category": "heart_rate",
                    "value": payload_data.get("value"),
                    "unit": payload_data.get("unit", "bpm"),
                    "observed_at": observed_at,
                    "source_plugin_id": "health-demo-sync",
                    "source_record_ref": raw_id,
                }
            )

    return observations
```

这里真正不能漏的是：

- `category`
- `value`
- `source_record_ref`

少任何一个，后端链路都会失败或者写不进记忆。

## 6. 第五步：先做静态自检

现在别急着调接口，先过 4 个最基本的检查：

1. `manifest.json` 的 `entrypoints` 路径能对上实际文件
2. `connector.py` 里有 `sync()`
3. `ingestor.py` 里有 `transform()`
4. `manifest.id`、目录名、模块路径能一眼对上

如果这 4 个都没过，后面调接口只会浪费时间。

## 7. 第六步：走一次完整同步调用

当前最适合验证这类插件的入口是：

- `POST /api/v1/ai-config/{household_id}/agents/{agent_id}/plugin-memory-checkpoint`

请求示例：

```json
{
  "plugin_id": "health-demo-sync",
  "payload": {
    "member_id": "member-001",
    "captured_at": "2026-03-13T07:30:00Z"
  },
  "trigger": "agent-checkpoint"
}
```

这个调用会自动做完下面这些事：

1. 调 `connector`
2. 保存原始记录
3. 调 `memory-ingestor`
4. 写入 Observation 记忆
5. 返回新的 Agent memory insight

## 9. 如果失败，先查哪里

最常见的失败点就这些：

### `manifest` 路径写错

症状：插件入口加载失败。

先查：

- `entrypoints` 模块路径
- 目录名和模块名
- 函数名是不是写错

### `memory-ingestor` 返回结构不对

症状：同步跑了，但写不进记忆。

先查：

- 有没有返回 `list`
- 每一项是不是 `dict`
- 有没有 `source_record_ref`
- 有没有 `category` 和 `value`

### `plugin_id` 对不上

症状：接口调用时提示插件不存在。

先查：

- 请求里的 `plugin_id`
- `manifest.json` 里的 `id`

## 10. 这篇 walkthrough 帮你确认了什么

如果你按这篇做通了，说明你已经掌握了当前版本最核心的接入方式：

- 用 `manifest` 声明插件
- 用 `connector` 返回原始记录
- 用 `memory-ingestor` 转标准 Observation
- 用统一接口把数据同步进项目

这才是当前版本真正可工作的插件接入路径。
