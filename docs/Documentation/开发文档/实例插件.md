# 实例插件

第一次做插件，别一上来就搞高风险控制或者复杂协议。先拿仓库里现成的内置插件看清套路。

## 示例 1：`health-basic-reader`

位置：

- `apps/api-server/app/plugins/builtin/health_basic/`

适合学什么：

- 最小 `integration` 插件长什么样
- 怎么声明 `schedule` trigger
- 怎么给首页提供 dashboard card

manifest 里的重点：

- `types: ["integration"]`
- `entrypoints.integration`
- `capabilities.integration`
- `dashboard_cards`

这类插件最适合当你的第一个参考样本，因为它不需要 discovery，也没有动作执行的复杂度。

## 示例 2：`builtin.provider.chatgpt`

位置：

- `apps/api-server/app/plugins/builtin/ai_provider_chatgpt/manifest.json`

适合学什么：

- `ai-provider` 的字段 schema 怎么写
- runtime capability 怎么声明
- OpenAI 兼容供应商的能力边界怎么表达

这类插件不是拿来学业务逻辑的，而是拿来学“宿主负责治理，插件负责适配”的分工。

## 示例 3：`channel-feishu`

位置：

- `apps/api-server/app/plugins/builtin/channel_feishu/manifest.json`

适合学什么：

- `config_specs` 怎么配
- `ui_schema` 怎么描述表单
- channel 绑定能力怎么声明

如果你要做通讯平台接入，这个例子比空写 README 有用得多。

## 示例 4：`builtin.theme.chun-he-jing-ming`

位置：

- `apps/api-server/app/plugins/builtin/theme_chun_he_jing_ming_pack/`

适合学什么：

- `theme-pack` 资源插件怎么声明
- `tokens_resource`、`platform_targets`、`resource_source` 怎么写

## 做第一个插件的推荐顺序

1. 先选简单类型：优先 `integration` 或 `theme-pack`
2. 先把 manifest 写对
3. 再写最小入口函数
4. 再把配置接上
5. 最后跑宿主内验证

## 一个最小 `integration` 示例骨架

```text
my_demo_plugin/
├── __init__.py
├── manifest.json
├── integration.py
├── README.md
└── tests/
```

`manifest.json` 至少应该有：

```json
{
  "id": "my-demo-plugin",
  "name": "我的示例插件",
  "version": "0.1.0",
  "api_version": 1,
  "types": ["integration"],
  "permissions": ["device.read"],
  "risk_level": "low",
  "triggers": ["manual"],
  "entrypoints": {
    "integration": "my_demo_plugin.integration.sync"
  },
  "capabilities": {
    "integration": {
      "domains": ["demo"],
      "instance_model": "single_instance",
      "refresh_mode": "polling",
      "supports_discovery": false,
      "supports_actions": false,
      "supports_cards": false,
      "entity_types": ["demo.entity"]
    }
  }
}
```

## 验证时先看什么

- 宿主能不能扫描到插件
- 插件状态页能不能看到它
- 配置能不能保存
- 执行入口能不能真正跑到
- 禁用后是否被统一拦截

## 不推荐的第一插件选择

- 高风险动作插件
- 需要复杂鉴权和回调的平台通道
- 需要大量宿主上下文写权限的插件

第一插件要先学结构，不是先秀复杂度。
