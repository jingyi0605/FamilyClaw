# 实例插件

## 这页要解决什么

- 给第一次做插件的人一条能照着走的最小路径

## 推荐示例路线

1. 先选一个最简单的插件目标，不要一上来就做高风险控制
2. 先写 `manifest.json`
3. 再写正式入口文件
4. 先做静态自检，再调接口

## 示例骨架

```text
health-demo-sync/
  manifest.json
  requirements.txt
  README.md
  plugin/
    __init__.py
    connector.py
    ingestor.py
```

## 完成标准

- 开发者知道第一个插件应该先做什么，而不是先写一堆业务代码
