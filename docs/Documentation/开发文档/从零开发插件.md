# 从零开发插件

这是一份最小可执行 walkthrough，只讲现在正确的开发路径。

## 第一步：把目录建对

假设我们要开发 `health-demo-sync`，源码目录应该建在：

- `apps/api-server/plugins-dev/health-demo-sync/`

不要建到：

- `apps/api-server/data/plugins/official/`

这是历史目录，已经废弃。

## 第二步：准备最小目录

```text
apps/api-server/plugins-dev/health-demo-sync/
  manifest.json
  requirements.txt
  README.md
  plugin/
    __init__.py
    integration.py
```

## 第三步：先写 `manifest.json`

最少要写清楚：

- `id`
- `name`
- `version`
- `types`
- `permissions`
- `risk_level`
- `entrypoints`

如果插件需要配置，再补 `config_specs`。

## 第四步：写入口代码

入口模块路径必须和 `manifest.json` 里声明的一致。

比如：

```json
{
  "entrypoints": {
    "integration": "plugin.integration.sync"
  }
}
```

那你就必须提供：

```python
def sync(payload=None):
    return {"ok": True, "payload": payload or {}}
```

## 第五步：先按开发覆盖源调试

这个阶段不用先安装：

- 把源码放进 `plugins-dev`
- 直接验证家庭插件注册表能看到它
- 验证执行入口和配置行为是否正确

如果当前家庭里已经装了同 `plugin_id` 的第三方插件，`plugins-dev` 会先覆盖它。

## 第六步：只有在验证安装链路时才安装

### 本地安装

- 目标目录是 `data/plugins/third_party/local/`

### 市场安装

- 目标目录是 `data/plugins/third_party/marketplace/`

无论哪种安装方式，都不要把开发源码直接塞进安装目录。

## 第七步：记住冲突规则

- `builtin > plugins-dev > 已安装 third_party`
- `plugins-dev` 可以覆盖第三方插件
- `plugins-dev` 不可以覆盖 `builtin`

## 最后检查

至少确认这几件事：

1. `manifest.json` 能通过校验
2. 入口模块能 import
3. 家庭插件注册表能看到插件
4. 同名第三方插件会被开发版覆盖
5. 删除安装态不会删掉 `plugins-dev` 源码
