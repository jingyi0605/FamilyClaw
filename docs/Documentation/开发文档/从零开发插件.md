# 从零开发插件

这页是 walkthrough，不是正式类型总表。

先把边界说清楚：

- 这篇示例沿用仓库里仍然存在的 reader 链路样例，目的是帮你看懂后台任务、原始记录和记忆写入怎么串起来。
- 新插件不要再把 `connector` / `memory-ingestor` 当成首选正式类型。
- 如果你是在做新插件，先按 [类型总表](./类型总表.md) 选 `integration`、`action`、`channel`、`ai-provider` 这些正式类型。

## 先选一个最简单的目标

这篇 walkthrough 选的目标是：

- 做一个最小的健康数据同步样例
- 用一次后台任务把步数和心率写进系统记忆

示例插件 id 用这个：

- `health-demo-sync`

## 第一步：放到正确目录

第三方插件推荐不要直接塞进主项目源码树。

先建自己的插件目录：

```text
health-demo-sync/
  manifest.json
  requirements.txt
  README.md
  plugin/
    __init__.py
    connector.py
    ingestor.py
```

仓库目录名可以自由一点，但 `manifest.id` 还是用中划线。

## 第二步：写 `manifest.json`

先放最小可运行版本：

```json
{
  "id": "health-demo-sync",
  "name": "健康演示同步插件",
  "version": "0.1.0",
  "types": ["connector", "memory-ingestor"],
  "permissions": [
    "health.read",
    "memory.write.observation"
  ],
  "risk_level": "low",
  "triggers": ["manual", "agent-checkpoint"],
  "entrypoints": {
    "connector": "plugin.connector.sync",
    "memory_ingestor": "plugin.ingestor.transform"
  },
  "description": "读取演示健康数据，并写成 Observation 记忆。",
  "vendor": {
    "name": "FamilyClaw Demo",
    "contact": "internal-demo"
  }
}
```

这里最容易犯的错：

- `id` 写成下划线
- `entrypoints` 模块路径和 `plugin/` 下真实目录对不上
- 声明了 `memory-ingestor`，但没配 `memory_ingestor` 入口

## 第三步：写 `plugin/connector.py`

这个样例先返回两条原始记录：步数和心率。

```python
from __future__ import annotations


def sync(payload: dict | None = None) -> dict:
    normalized_payload = payload or {}
    member_id = normalized_payload.get("member_id", "demo-member")
    captured_at = normalized_payload.get("captured_at", "2026-03-13T07:30:00Z")

    return {
        "source": "health-demo-sync",
        "mode": "connector",
        "received_payload": normalized_payload,
        "records": [
            {
                "record_type": "steps",
                "external_member_id": member_id,
                "member_id": member_id,
                "value": 9032,
                "unit": "count",
                "captured_at": captured_at
            },
            {
                "record_type": "heart_rate",
                "external_member_id": member_id,
                "member_id": member_id,
                "value": 68,
                "unit": "bpm",
                "captured_at": captured_at
            }
        ]
    }
```

这个返回值满足当前样例链路最关心的东西：

- 有 `records`
- 每条 record 都有 `record_type`
- 有 `value`
- 有 `captured_at`

## 第四步：写 `plugin/ingestor.py`

现在把原始记录转成 Observation。

```python
from __future__ import annotations


def transform(payload: dict | None = None) -> list[dict]:
    normalized_payload = payload or {}
    records = normalized_payload.get("records", [])
    observations: list[dict] = []

    for record in records:
        if not isinstance(record, dict):
            continue

        raw_id = record.get("id")
        payload_data = record.get("payload", {})
        if not isinstance(payload_data, dict):
            payload_data = {}

        record_type = record.get("record_type")
        subject_id = payload_data.get("member_id") or payload_data.get("external_member_id")
        observed_at = record.get("captured_at")

        if record_type == "steps":
            observations.append(
                {
                    "type": "Observation",
                    "subject_type": "Person",
                    "subject_id": subject_id,
                    "category": "daily_steps",
                    "value": payload_data.get("value"),
                    "unit": payload_data.get("unit", "count"),
                    "observed_at": observed_at,
                    "source_plugin_id": "health-demo-sync",
                    "source_record_ref": raw_id,
                }
            )
        elif record_type == "heart_rate":
            observations.append(
                {
                    "type": "Observation",
                    "subject_type": "Person",
                    "subject_id": subject_id,
                    "category": "heart_rate",
                    "value": payload_data.get("value"),
                    "unit": payload_data.get("unit", "bpm"),
                    "observed_at": observed_at,
                    "source_plugin_id": "health-demo-sync",
                    "source_record_ref": raw_id,
                }
            )

    return observations
```

这里真正不能漏的是：

- `category`
- `value`
- `source_record_ref`

少任何一个，后端链路都会失败或者写不进记忆。

## 第五步：先做静态自检

现在别急着调接口，先过 4 个最基本的检查：

1. `manifest.json` 的 `entrypoints` 路径能对上实际文件
2. `connector.py` 里有 `sync()`
3. `ingestor.py` 里有 `transform()`
4. `manifest.id`、目录名、模块路径能一眼对上

## 第六步：创建一次后台任务并跟到终态

当前最适合验证这类存量 reader 样例的入口是：

- `POST /api/v1/plugin-jobs`

请求示例：

```json
{
  "plugin_id": "health-demo-sync",
  "plugin_type": "connector",
  "payload": {
    "member_id": "member-001",
    "captured_at": "2026-03-13T07:30:00Z"
  },
  "trigger": "manual"
}
```

这个调用不会同步把整条链直接跑完，它会先做下面这些事：

1. 创建 `plugin_job`
2. 快速返回 `job_id` 和当前状态
3. 后台 worker 再去执行 `connector`
4. 保存原始记录
5. 调 `memory-ingestor`
6. 写入 Observation 记忆

## 第七步：你应该看到什么结果

成功时至少应该看到这些现象：

- 创建接口先返回 `queued = true`
- 能拿到 `job_id`
- worker 执行完成后，`job.status = succeeded`
- 最新尝试 `latest_attempt.status = succeeded`
- 如果是数据插件，后续能看到原始记录和记忆写入结果都对得上

## 如果失败，先查哪里

### `manifest` 路径写错

症状：插件入口加载失败。

先查：

- `entrypoints` 模块路径
- 目录名和模块名
- 函数名是不是写错

### `memory-ingestor` 返回结构不对

症状：同步跑了，但写不进记忆。

先查：

- 有没有返回 `list`
- 每一项是不是 `dict`
- 有没有 `source_record_ref`
- 有没有 `category` 和 `value`

### `plugin_id` 对不上

症状：接口调用时提示插件不存在。

先查：

- 请求里的 `plugin_id`
- `manifest.json` 里的 `id`

## 这篇 walkthrough 真正帮你确认了什么

如果你按这篇做通了，说明你已经掌握了两件事：

1. 存量 reader 链路在当前仓库里到底怎么跑。
2. 后台任务、原始记录、记忆写入之间的关系是什么。

但再强调一次：

- 这不是新插件正式类型推荐。
- 新插件的设计边界，仍然以 [类型总表](./类型总表.md) 和 [对接方式](./对接方式.md) 为准。
