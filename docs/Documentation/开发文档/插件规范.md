# 插件规范

## 这页要解决什么

- 用人话讲清楚宿主和插件的分工边界

## 当前稳定口径

- 宿主保留平台规则、统一入口、权限、审计、调度和标准数据语义
- 插件负责接第三方 API、设备或平台，并把外部字段映射成宿主标准 DTO
- 插件产出的设备、实体、卡片和动作结果必须已经是最终标准结果，宿主不再做领域补丁

## 当前正式插件类型

- `integration`
- `action`
- `agent-skill`
- `channel`
- `region-provider`
- `ai-provider`
- `locale-pack`
- `theme-pack`

## 当前独占槽位

- `memory_engine`
- `memory_provider`
- `context_engine`

- 开发者知道哪些事该放宿主，哪些事必须放插件
