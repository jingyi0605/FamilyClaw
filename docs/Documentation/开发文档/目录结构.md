# 目录结构

## 最小目录模板

```text
your_plugin/
  manifest.json
  requirements.txt
  README.md
  __init__.py
  integration.py
  action.py
  channel.py
  agent_skill.py
  ai_provider.py
  memory_engine.py
  memory_provider.py
  context_engine.py
  locales/
  tests/
```

## 硬规则

- 一个插件目录只放一个 `manifest.json`
- 一个 `manifest.json` 只描述一个插件
- 目录名和 `manifest.id` 至少要能一眼对上

## 文件职责

- `manifest.json`：插件声明
- `requirements.txt`：插件自己的依赖
- `README.md`：给审核和维护者看的最小说明
- 各入口文件：按插件声明的正式类型决定是否存在

## 参考来源

- `04-插件目录结构规范.md`

## 完成标准

- 插件开发者知道文件该怎么摆，而不是自己发明目录层级
