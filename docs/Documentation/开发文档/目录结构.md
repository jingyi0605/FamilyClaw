# 目录结构

这页不讲套话，只讲现在仓库里最值得记住的目录。

## 项目级目录

```text
FamilyClaw/
├── apps/
│   ├── api-server/
│   ├── user-app/
│   ├── open-xiaoai-gateway/
│   ├── start-api-server.sh
│   └── start-open-xiaoai-gateway.sh
├── packages/
│   ├── user-core/
│   ├── user-platform/
│   └── user-ui/
├── docs/
│   ├── Documentation/
│   ├── 开发设计规范/
│   └── 开发者文档/
├── docker/
├── specs/
├── package.json
└── Dockerfile
```

## 各目录到底干什么

### `apps/api-server`

后端主服务。

重点看：

- `app/main.py`
- `app/api/v1/`
- `app/modules/`
- `app/core/`
- `migrations/`

### `apps/user-app`

前端主应用，基于 Taro。

重点看：

- `src/app.config.ts`
- `src/pages/`
- `src/runtime/`
- `config/platform/h5.ts`

### `apps/open-xiaoai-gateway`

语音网关。

重点看：

- `open_xiaoai_gateway/settings.py`
- `apps/start-open-xiaoai-gateway.sh`

### `packages/user-core`

前端共享类型和 API client 事实源。

重点看：

- `src/domain/types.ts`
- `src/api/create-api-client.ts`
- `src/services/`

### `packages/user-platform`

平台能力适配层，比如存储、实时能力等。

### `packages/user-ui`

UI 组件与主题归一化层。

### `docs/Documentation`

官方文档站源码。改用户可见行为、安装方式、接口语义时，这里必须同步更新。

### `docs/开发设计规范`

仓库里仍存在的补充资料目录，但它不是当前官方文档站的主导航入口。

如果你是按 VitePress 文档站阅读和跳转，开发规则请优先看当前站内这些页面：

- [环境准备](./环境准备.md)
- [后端开发](./后端开发.md)
- [插件规范](./插件规范.md)
- [字段规范](./字段规范.md)
- [对接方式](./对接方式.md)

### `docs/开发者文档`

仓库里仍存在的专题资料目录，但当前官方开发手册已经收敛到 `docs/Documentation/开发文档/`。

如果你需要正式可跳转、可持续维护的开发文档，直接看当前目录下这些页面，不要把理解建立在站外目录上。

## 插件目录长什么样

内置插件都在：

- `apps/api-server/app/plugins/builtin/`

典型目录示例：

```text
apps/api-server/app/plugins/builtin/health_basic/
├── __init__.py
├── integration.py
├── ingestor.py
└── manifest.json
```

另一个带配置的通道插件示例：

```text
apps/api-server/app/plugins/builtin/channel_feishu/
├── __init__.py
├── channel.py
└── manifest.json
```

主题插件示例：

```text
apps/api-server/app/plugins/builtin/theme_chun_he_jing_ming_pack/
├── manifest.json
└── themes/
```

## 新插件最小目录模板

```text
your_plugin/
├── manifest.json
├── __init__.py
├── README.md
├── requirements.txt          # 需要额外依赖时再放
├── integration.py            # integration 类型才需要
├── action.py                 # action 类型才需要
├── channel.py                # channel 类型才需要
├── locales/                  # 有额外文案时再放
└── tests/                    # 建议保留
```

## 目录设计的坏味道

看到下面这些情况，通常说明结构已经开始烂了：

- 一个插件目录里塞了多个 `manifest.json`
- 插件目录名和 `manifest.id` 完全对不上
- 把宿主业务逻辑写进插件目录
- 把插件私有模型塞回宿主总模型入口

结构不清楚，后面所有文档和排障都会一起烂。
