# 对接方式

## 当前主链路

### integration

1. 宿主读取 manifest
2. 校验类型、入口和能力声明
3. 注入运行时上下文和正式配置
4. 调用 `integration.refresh`
5. 插件返回标准结果
6. 宿主统一校验、去重、落库和更新状态

### action

1. 校验插件已启用
2. 校验权限和风险级别
3. 必要时走人工确认
4. 调动作入口

### ai-provider

1. 宿主找到对应插件
2. 读取 `entrypoints.ai_provider`
3. 调统一 driver 入口
4. 宿主统一处理权限、审计、密钥和结果收口

## 硬边界

- 插件执行入口只吃 JSON 可序列化 payload
- 不要依赖请求对象、ORM Session 或不可序列化对象
- 插件返回的实体和卡片必须是标准 DTO，不要发明私有协议
