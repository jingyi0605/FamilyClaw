# 字段规范

## 最小 manifest 必填字段

- `id`
- `name`
- `version`
- `api_version`
- `types`
- `permissions`
- `entrypoints`
- `capabilities`

## 需要配置时再补

- `config_specs`
- `locales`

## 当前要点

- `enum_options` 和 `option_source` 不能同时写
- `depends_on` 和 `clear_on_dependency_change` 用来表达字段联动
- 面向用户的文案建议保留原文，同时可补词典 key

## 参考来源

- `03-manifest字段规范.md`

## 完成标准

- 开发者知道哪些字段是硬要求，哪些是扩展项
