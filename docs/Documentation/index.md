## 先看哪一类文档

- 如果你是第一次接触 FamilyClaw，请从 [快速开始](/快速开始/文档总览) 进入。
- 如果你的目标是把FamilyClaw尽快跑起来，请直接打开 [安装部署](/安装部署/概览)。
- 如果你已经装好本产品FamilyClaw，想知道页面怎么用，请看 [使用手册](/使用指南/首次登录与初始化)。
- 如果你要改代码、接接口、写插件，请看 [开发手册](/开发文档/环境准备)。

## 这套文档的写法原则

- 用户文档优先解决“下一步该点什么、该填什么、哪里容易错”。
- 开发文档优先解决“真实入口在哪、边界在哪、不能乱碰什么”。
- 仓库里没有的功能、命令、部署方式，不会写进正式文档。
- 英文文档继续保留在 `/en/`，中文根路径是当前主维护入口。

## 当前文档结构

- `快速开始`：先认识产品，再选安装、使用还是开发路径。
- `安装部署`：覆盖 Docker、源码、NAS、Ubuntu、Windows 五条常见路径。
- `使用手册`：覆盖首次登录、仪表盘、家庭、助手、记忆、设置、插件等实际使用页面。
- `开发手册`：覆盖开发环境、后端边界、插件体系、接口联调和现行规范。
- `沟通交流`：保留官网和社群入口。
