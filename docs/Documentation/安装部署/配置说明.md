# 配置说明

单容器部署时，优先用环境变量改配置，不要进容器里手改文件。镜像里的默认值只适合本地试跑，不适合正式环境。

## 配置来源

按优先级从高到低：

1. `docker run -e ...` 或编排平台注入的环境变量
2. 镜像内脚本根据环境变量生成的运行时配置
3. 应用源码里的默认值

单容器路径下，用户主要只需要改环境变量，不需要编辑容器内的 `.env` 文件。

## 必填配置

| 变量 | 作用 | 建议 |
| --- | --- | --- |
| `FAMILYCLAW_DB_PASSWORD` | PostgreSQL 业务账户密码 | 生产环境必须替换默认值 |
| `FAMILYCLAW_VOICE_GATEWAY_TOKEN` | `api-server` 和语音网关共用的鉴权令牌 | 启用语音网关时必须替换 |

## 常用可选配置

| 变量 | 默认值 | 用途 |
| --- | --- | --- |
| `FAMILYCLAW_DB_NAME` | `familyclaw` | PostgreSQL 数据库名 |
| `FAMILYCLAW_DB_USER` | `familyclaw` | PostgreSQL 业务账户名 |
| `FAMILYCLAW_DB_PORT` | `5432` | 容器内 PostgreSQL 监听端口 |
| `FAMILYCLAW_API_PORT` | `8000` | `api-server` 容器内监听端口 |
| `FAMILYCLAW_GATEWAY_PORT` | `4399` | 语音网关容器内监听端口 |
| `FAMILYCLAW_NGINX_PORT` | `8080` | Nginx 容器内监听端口 |
| `FAMILYCLAW_ENABLE_GATEWAY` | `1` | 是否启动 `open-xiaoai-gateway` |
| `FAMILYCLAW_BUILD_CHANNEL` | `development` | 构建通道，建议发布镜像时改成 `stable` 或 `preview` |
| `FAMILYCLAW_RELEASE_URL` | 空 | 版本详情页链接，会展示在版本信息接口里 |

## 固定目录约定

这些路径建议固定，不要改成一堆碎片目录：

- `/data/postgres`：数据库数据
- `/data/plugins`：插件及相关持久化文件
  - 手动 ZIP 安装插件当前落在 `/data/plugins/third_party/manual/<household_id>/<plugin_id>/<version>--<timestamp>--<id>/`
  - 插件市场安装插件会落在对应来源的 marketplace 目录
  - 这两个目录更新后，插件列表和后续执行会直接识别当前挂载，不需要靠重启后端服务补状态
  - 但这不是 Python 模块热替换；已经在跑的旧任务仍按旧版本收口
- `/data/backups`：升级前备份
- `/data/runtime`：当前安装状态和诊断文件
- `/data/voice-runtime-artifacts`：语音运行时产物

## 改完配置怎么验证

1. `docker exec familyclaw familyclawctl doctor`
2. `docker exec familyclaw familyclawctl version`
3. `curl -fsS http://127.0.0.1:8080/api/v1/system/version`

如果你改的是数据库或网关相关变量，至少再重启一次容器确认它能重新拉起。

## 常见配置错误

- 改了 `FAMILYCLAW_DB_PASSWORD`，但没有用同一套密码重启新容器
- 关闭网关时只停掉 `4399` 端口映射，没有设置 `FAMILYCLAW_ENABLE_GATEWAY=0`
- 把发布环境也保留成 `development` 通道，导致版本展示和发布语义不一致
