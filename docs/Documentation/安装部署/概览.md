# 安装部署概览

## 这页要解决什么

- 帮你判断应该走哪一种安装或部署路径

## 路径选择

- 想最快跑起来：看 [Docker安装](./Docker安装.md)
- 想自己改代码和调试：看 [源码安装](./源码安装.md)
- 想放在 NAS：看 [NAS部署](./NAS部署.md)
- 想用 Linux 服务器：看 [Ubuntu部署](./Ubuntu部署.md)
- 想在 Windows 环境直接跑：看 [Windows部署](./Windows部署.md)

## 选择原则

- 能用 Docker 就别先折腾源码
- 需要长期维护或二次开发，再走源码安装
- NAS、Ubuntu、Windows 都是环境差异，不应该引出三套完全不同的产品逻辑

## 完成标准

- 读者能判断自己应该打开哪一页，而不是每页都扫一遍
