# AI配置与插件使用
> 这页只讲用户能看到的配置入口和使用方式，不展开内部实现。

## 这页要解决什么
- 说明 AI 供应商怎么配
- 说明插件启用状态会怎样影响模型可用性
- 说明 Agent 设置里“文本”和“意图识别”两条链路该怎么分开配

## AI 供应商配置入口
- 进入“设置 / AI”，先配置家庭可用的 AI 供应商档案。
- 常见做法是保留一个稳定的文本主模型，再按需要补充视觉、音频或图片能力对应的模型。
- 如果只是想调整聊天效果，不需要改插件代码，直接在这里切换模型档案即可。

## 插件查看与启用方式
- 进入“设置 / 插件”可以查看当前家庭已经挂载的插件，以及每个插件是否启用。
- 插件市场安装成功后，插件会立刻进入当前家庭的插件列表，不需要重启后端服务才能被识别。
- 运行时安装进来的插件默认不会自动启用。市场插件还要先完成配置，才能手动启用。
- 启用后的后续执行会直接按当前挂载生效，但这不等于把已经导入到 Python 进程里的旧模块热替换。
- 手动 ZIP 安装和市场安装的插件，都可以在插件详情页直接删除，不需要重启后端服务。
- 内置插件不能删除。
- 如果这个插件还被集成实例、通道账号、设备绑定或集成发现记录引用，删除会被直接阻止，必须先把这些引用清掉。
- AI 供应商插件被禁用后，相关模型档案和路由会一起失效。
- 如果页面里突然选不到某个模型，先检查对应插件是不是被停用了。

## Agent 模型绑定
- Agent 设置页支持给当前 Agent 单独指定模型，不会影响家庭里其他 Agent。
- “文本”适合普通慢链路回复，例如自由聊天、总结、整理说明。
- “意图识别”适合 15 秒敏感链路，例如先判断这句话是闲聊、问答、提醒还是设置修改。
- 如果你希望意图识别更快，可以把“意图识别”单独绑到 Qwen；把“文本”继续留给 GLM 这类慢一些但更稳的模型。
- 某一项留空时，系统会继续沿用家庭级能力路由，不需要每个 Agent 都重新配一遍。

## 常见使用场景
- 家庭默认问答：给“文本”绑定日常主模型。
- 快速意图判断：给“意图识别”绑定响应更快的模型。
- 某个专用 agent-skill 需要独立模型：在 Agent 设置页的 `agent-skill` 专属模型里单独指定。

## 失败时先看哪里
- 先看对应 AI 供应商插件是否启用。
- 再看家庭级能力路由或 Agent 单独模型绑定是不是留空、指错，或者引用了已禁用的模型档案。
- 如果只有“意图识别”这一项异常，优先检查 Agent 设置页里的“意图识别”绑定是否还在；留空时会回退到家庭级文本路由。

## 完成标准
- 用户能完成一次供应商配置，并能分开理解“文本”与“意图识别”这两条链路该怎么配
- 用户能在 Agent 设置页看到并保存“意图识别”模型绑定
- 留空单独绑定时，系统仍能沿用家庭默认路由继续工作
