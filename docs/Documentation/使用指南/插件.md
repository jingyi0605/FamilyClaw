# 插件

插件页适合用来给 FamilyClaw 加新能力，或者管理已经接进来的能力。

如果你想接入新的 AI 服务、通讯渠道、主题，或者其他扩展功能，通常都会从这里开始。

![fc-doc-20260320T000611.webp](assets/fc-doc-20260320T000611.webp)

## 这里可以做什么

- 浏览可用插件，查看它们来自哪里、当前是什么状态、适不适合你使用。
- 在卡片视图和列表视图之间切换。
- 打开插件详情，查看介绍、风险提示、版本状态和提供方信息。
- 进入插件市场，安装、升级、回滚或卸载第三方插件。
- 管理插件源并按需同步目录。
- 启用或禁用插件。

## 一般怎么用

1. 打开插件页，先确认右上角家庭选择正确。
2. 想找新插件时，先打开“插件市场”。
3. 想看细节时，点开插件详情。
4. 想启用或停用插件时，直接在列表或详情里操作。
5. 如果插件需要配置，再去设置页补齐配置。

## 页面上的标识怎么理解

- **来源标识**：区分这是内置插件还是第三方插件。
- **开发版 / 已安装版**：当同一个 `plugin_id` 同时存在开发版和已安装版时，页面会把两个变体都显示出来，不会把其中一个偷偷藏掉。
- **开发插件生效中**：看到这个标签，表示当前家庭实际运行的是 `plugins-dev` 里的开发版，不是安装目录里的版本。
- **版本状态**：告诉你当前版本是不是已安装、能不能升级、有没有被阻止。
- **风险等级**：风险越高，越应该先看清说明再启用。

## 同名开发版和已安装版怎么切换

如果某个第三方插件同时存在开发版和已安装版：

- 页面会同时显示两个变体。
- 同一时刻只允许启用其中一个。
- 默认优先开发版。
- 想测试正式安装版时，直接切换启用到已安装版即可，不需要手动移动 `plugins-dev` 目录。

## 常见问题

- **启用失败**
  先看这个插件要求的配置是不是还没填完。

- **为什么我同时看到了两个同名插件**
  这通常表示系统里同时存在开发版和已安装版。现在这是正常行为，不是重复数据。

- **为什么页面写着“开发插件生效中”**
  因为当前实际运行的是 `plugins-dev` 里的开发版。想测试安装版，直接切换启用即可。

- **为什么内置插件不能被开发版替换**
  因为 `builtin` 始终最高优先级，开发版不能覆盖它。
