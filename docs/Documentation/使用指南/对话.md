# 对话 / 助手

对话页是你和 FamilyClaw 最直接的沟通入口。

如果你已经完成登录和初始化，并且家里已经配置好至少一个可用的 AI 助手，就可以从这里开始聊天。

当前真正可以使用的是 `个人聊天` 标签页；`家人圈` 还在开发中。

![fc-doc-20260320T000734.webp](assets/fc-doc-20260320T000734.webp)

## 你可以在这里做什么

### 1. 新建会话并管理历史会话

进入页面后，你可以直接开始新的对话，也可以切换到以前的会话继续聊。

当前页面支持：

- 点击标签栏右侧的 `新对话` 按钮创建新会话
- 点击标签栏中间的会话标题展开会话历史列表
- 查看每个会话的标题、最近一条消息摘要、最近活跃时间
- 点击某个历史会话，继续回到这段上下文里对话
- 在历史会话右侧直接删除不再需要的旧会话

如果你第一次进入页面、当前还没有任何会话，页面会直接给出“新建会话”的入口，不需要你先做别的复杂操作。

如果一段会话还在生成回复，删除按钮会暂时受限。等当前回复结束后再删，避免把还在运行的对话过程硬砍掉。

![fc-doc-20260320T000759.webp](assets/fc-doc-20260320T000759.webp)

### 2. 选择不同的 AI 助手来对话

如果你的家庭里配置了多个可对话助手，这个页面允许你切换当前助手。

页面会根据当前会话自动使用对应助手。移动端顶部仍然会显示当前助手名称；当存在多个可切换助手时，你可以点击头像区域进行切换。切换后，页面会基于新助手创建一段新的会话，避免把不同助手的上下文混在一起。

只有当前已启用、并且支持对话的助手，才会出现在这里。
如果你切换了助手，通常会从一段新的会话开始，这样不会把不同助手的上下文混在一起。

### 3. 发送消息并实时接收回复

你可以：

- 在输入框中输入问题
- 按 `Enter` 直接发送
- 按 `Shift + Enter` 换行
- 等待助手实时返回内容
- 通过右上角的连接信号图标查看当前 WebSocket 状态

消息会逐步返回，所以你通常会看到：

- 你的消息先出现在会话中
- 助手消息进入“正在生成”状态
- 内容逐步返回并最终完成

为了减少长回复时的闪烁和跳变，流式生成阶段会先按稳定文本持续追加，等这一条消息真正完成后，再一次性按 Markdown 规则渲染最终版。这样既能保持阅读位置稳定，也不会把最终格式能力拿掉。

为了避免模型输出里混入重复空行把消息撑得很散，页面会在展示前自动把连续重复空白压成一个正常的段落换行。正常段落、列表和代码块的基本结构会保留，不需要你手动整理格式。

如果实时连接中途抖动，页面会自动尝试重连，并在右上角的信号图标里显示“连接中 / 重连中 / 已连接”这类状态。正常情况下不需要你手动刷新页面。

点击右上角的信号图标，还可以直接看到当前连接状态说明；如果自动恢复太慢，面板底部也提供了 `手动重连` 按钮，方便你立即重建当前会话的实时连接。

### 4. 直接使用建议问题，少打字

页面会自动拉取一组适合当前家庭和当前助手的建议问题。

这些建议问题会出现在两个地方：

- 页面上下文面板中的快捷提问区
- 某些助手回复下面的建议追问按钮

你可以直接点这些问题继续聊，适合：

- 不知道该怎么开口时
- 想快速试试助手能力时
- 想围绕当前话题继续追问时

这对第一次使用的人特别友好，因为你不需要先想好怎么组织提问。
![fc-doc-20260320T000904.webp](assets/fc-doc-20260320T000904.webp)

### 5. 查看当前对话上下文

标签栏右侧有 `会话详情` 入口，打开后会看到右侧上下文面板。

这个面板目前会显示：

- 当前家庭
- 当前使用的助手
- 当前待处理动作数量
- 最近记忆
- 最近动作记录
- 快捷提问按钮

其中“最近记忆”会优先显示当前会话里最近提取到的事实；如果暂时没有可展示的事实，页面会回退显示建议问题。

这个面板主要是帮你快速理解：

- 助手此刻是在为哪个家庭服务
- 它最近参考了哪些上下文
- 有没有待确认的动作或建议

![fc-doc-20260320T000844.webp](assets/fc-doc-20260320T000844.webp)

### 6. 在对话里确认或忽略助手提案

当助手判断某件事不应该直接替你执行，而是需要你确认时，页面会在消息下方显示提案卡片。你可以直接在卡片里操作，不需要跳出页面。

目前真实支持的提案类型包括但不限于：

- 计划任务创建
- 计划任务更新
- 计划任务暂停
- 计划任务恢复
- 计划任务删除
- 提醒创建
- 配置应用
- 记忆写入类建议

常见操作方式：

- 点主按钮确认执行
- 点次按钮先忽略或稍后再说

如果某个计划任务提案关键信息还没补齐，确认按钮会受限，不会装作已经能执行。
![fc-doc-20260320T001006.webp](assets/fc-doc-20260320T001006.webp)


### 7. 确认动作、查看结果，必要时撤销

除了“先问你要不要做”的提案，你还会看到已经执行过或准备执行的动作记录。

当前页面里，动作可能以不同策略出现：

- 需要确认后再执行
- 只通知你，不拦截主流程
- 自动执行，但保留结果和撤销入口

你在页面上能看到的真实操作包括：

- 确认动作
- 忽略动作
- 撤销已完成动作（仅在该动作本身支持撤销时出现）

![fc-doc-20260320T001108.webp](assets/fc-doc-20260320T001108.webp)

### 8. 从对话直接跳到相关页面

当助手回复完成后，消息下方可能会给出快捷按钮，帮助你继续处理相关事项。

当前真实可跳转的页面包括：

- 家庭页
- AI 设置页
- 记忆页

这很适合下面这类场景：

- 助手建议你先去补家庭资料
- 助手提示你当前没有可用 AI 助手，需要先去设置页
- 助手提到某条记忆，需要你去记忆页进一步查看

## 第一次使用，建议这样试

1. 先确认你已经选中了正确的家庭，并且家庭里已经配置好至少一个 AI 助手。
2. 进入 `个人聊天` 标签页，点击 `新对话`。
3. 先点一条建议问题，确认助手能正常回复。
4. 再自己输入一个真实问题，观察是否有提案卡片或动作卡片出现。
5. 如有需要，打开右侧详情面板，确认当前上下文、最近记忆和最近动作。
6. 如果助手提示你去补配置，再从消息下方快捷跳转到对应页面处理。

## 常见问题

### 为什么我看到“没有可用助手”？

通常是因为当前家庭还没有可用的 AI 助手。先去设置页确认是否已经配置并启用。

### 为什么发送按钮有时点不了？

通常是因为连接还没准备好、刚切换会话，或者网络暂时不稳定。等一会儿再试通常就能恢复。

### 为什么有时删不了当前会话？

如果这段会话还在生成回复，页面会先拦住删除。原因很简单：后台这时还在处理这轮对话，强删只会把状态搞乱。等回复结束后再删就行。

### 为什么我会看到“AI 服务认证失败，请检查 API Key 是否正确”？

这不是对话页本身坏了，而是当前会话绑定的 AI 供应商认证没过。最常见的原因就是：

- API Key 填错了
- 供应商密钥已经失效、被撤销或被替换了
- 代理网关改写了认证头，导致上游把请求当成未授权

遇到这种提示时，直接去 `设置 -> AI 配置` 检查当前供应商的 API Key、Base URL 和代理配置，不要反复重试同一轮对话。

### 为什么我切换助手后像是重新开始聊了？

这是正常的。切换助手后，会以新的助手重新开始一段会话，避免上下文混在一起。

### 为什么有时只有回复，没有“确认”“忽略”这些按钮？

因为这些按钮只会在需要你确认或处理时才出现，不是每条回复都会有。

### 为什么我点进了 `家人圈`，但不能正常聊天？

因为当前真正可以使用的是 `个人聊天` 标签页，`家人圈` 还在开发中。

## 消息格式显示

现在助手消息会直接渲染常见 Markdown，不再把 `#`、`##` 这类标题当普通文本显示。当前已经覆盖：

- `#` 到 `######` 标题
- 粗体、斜体、删除线、行内代码、代码块
- 无序列表、有序列表、任务列表
- 引用、分隔线、表格
- 链接和图片

消息段落里的单个换行也会继续保留，比较符合聊天场景下 AI 回复的实际写法。

如果是一条还没结束的流式回复，页面会先按原始文本稳定显示，避免长消息在生成过程中反复重排；等回复完成后，再切回最终 Markdown 效果。
