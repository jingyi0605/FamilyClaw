# 设置

设置页适合用来把 FamilyClaw 调整成更符合你家使用习惯的样子。

像主题、语言、时区、AI 服务这些内容，基本都会从这里调整。

页头下面是一排分类标签，先点到对应分类，再看下面的具体内容。

![fc-doc-20260320T000409.webp](assets/fc-doc-20260320T000409.webp)

## 这里主要能改什么

- **外观与主题**：可以切换界面风格，选一个你看着更舒服的主题。
- **语言与时区**：会影响页面语言和时间显示，建议在第一次使用时就先设好。
- **设备与集成**：给接入插件创建实例，实例名称默认不会替你填好。每个插件可以自己声明建议占位，系统只负责展示，不再擅自拿插件名顶上去。
- **AI 设置入口**：如果你要配置模型、助手或其他 AI 相关能力，也从这里进入。

## AI 设置里可以做什么

- 创建或调整家庭管家和其他助手角色。
- 配置你要连接的 AI 服务，填写地址、密钥和模型名称等信息。
- `ChatGPT` 供应商现在同时支持 `Responses` 和 `Chat Completions` 两种协议，默认自动选择；如果网关只暴露旧兼容路由，会自动退回。
- `ChatGPT` 的 `base_url` 可以直接填 API 地址，也可以只填站点根地址；如果你填的是 `https://example.com`，系统会自动补成 `https://example.com/v1` 再发请求。
- 为不同用途选择合适的模型，比如聊天用哪个、其他任务优先用哪个。
- 打开某个模型服务的详情后，可以直接在底部关闭、删除或继续编辑；删除前系统会先让你确认一次。
- 如果初始化还没做完，这里也会告诉你还差哪些步骤。

### 如果你想接本地模型

现在内置支持这些本地模型供应商插件：

- `Ollama`
- `LM Studio`
- `LocalAI`

另外也支持很多常见的云端 AI 服务。

最省事的配置顺序通常是：

1. 先选供应商插件
2. 填本地服务地址
3. 如果你的本地服务启用了 API Key，再把 API Key 填上
4. 页面会自动刷新模型列表
5. 直接从列表里选模型；如果你要填自定义模型名，也可以继续手动输入

这样配完以后，你就可以直接在系统里使用本地模型了。

![fc-doc-20260320T000538.webp](assets/fc-doc-20260320T000538.webp)

## 使用时注意

- 页面顶部会显示当前家庭；切换家庭后，语言/主题/时区等设置随之切换。
- 如果当前家庭还没初始化完成，页面会提示你先回去完成初始化。

## 常见问题

- **主题缺失或突然失效**：先看看对应主题插件是不是被禁用了。
- **语言或时区没变化**：刷新页面或者重新登录一次，通常就会生效。
- **新建集成时实例名称怎么是空的**：这是现在的默认行为。输入框里的建议文案由插件自己声明；如果插件没声明，就只显示通用占位。最终名称还是要你自己确认后填写。
- **AI 服务保存失败**：先检查必填项是不是没填完整，再确认相关插件是不是已经启用。
