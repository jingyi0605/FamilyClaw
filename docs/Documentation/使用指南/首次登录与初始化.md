# 首次登录与初始化

## 适用场景

- 新部署后第一次进入系统。
- 新增家庭需要重新跑初始化向导。

## 登录入口

1. 浏览器访问部署地址（默认 `http://<主机>:8080`
2. 未登录会被重定向到登录页面。
3. 如果你是第一次登录，默认初始账号：user，密码：user；完成初始化流程后，会自动创建新的账号密码，user账号将被禁用。

![fc-doc-20260319T231835.webp](assets/fc-doc-20260319T231835.webp)

## 初始化向导步骤

向导入口 `/pages/setup/index`，Guard 会在家庭未完成初始化时自动跳转。
![fc-doc-20260319T234131.webp](assets/fc-doc-20260319T234131.webp)

1. **家庭资料 (`family_profile`)**
   - 填写家庭名称、时区、语言、地区（国家/省/市/区）。
![fc-doc-20260319T234213.webp](assets/fc-doc-20260319T234213.webp)
2. **首位成员 (`first_member`)**
   - 输入姓名/昵称/角色（默认管理员），可自动生成用户名；需要设置密码。
![fc-doc-20260319T234248.webp](assets/fc-doc-20260319T234248.webp)
3. **模型服务 (`provider_setup`)**
   - 选择已安装的 AI 供应商适配器（如 ChatGPT/OpenAI 兼容网关）。
   - 填写 `base_url`、`secret_ref`、`model_name`、隐私级别、超时等。
![fc-doc-20260319T234425.webp](assets/fc-doc-20260319T234425.webp)
支持众多AI提供商，包括本地的Ollama、LM Studio、LocalAI等
![fc-doc-20260319T234413.webp](assets/fc-doc-20260319T234413.webp)
4. **首个管家 (`first_butler_agent`)**
![fc-doc-20260319T234556.webp](assets/fc-doc-20260319T234556.webp)
   - 通过LLM对话，生成家庭管家的身份、性格、服务焦点，可在后续设置中再调整。
5. **完成 (`finish`)**
![fc-doc-20260319T234658.webp](assets/fc-doc-20260319T234658.webp)
   - 所有步骤完成后，会自动跳转仪表盘。

## 重新进入向导

- 若中途退出，可在重新登录后自动继续；向导会显示当前进度。
- 如果FamilyClaw认为家庭已完成初始化，将自动跳回仪表盘。

## 常见问题

- **无法进入向导/循环跳转**：清理浏览器缓存后重试；确认后端健康 `/api/v1/healthz`。
- **供应商表单缺字段**：确保你的baseurl和apikey正确，如果实在不可用，可以通过Github提issues。
- **完成后仍提示未初始化**：检查当前家庭上下文是否切换成功；必要时重新登录。

## 完成标准

- 至少完成家庭、首位成员、模型供应商和首个管家的创建。
- 登录后能进入仪表盘且不再被重定向到向导。
