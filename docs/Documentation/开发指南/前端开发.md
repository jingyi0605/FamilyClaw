## 建议内容结构

## 前端代码放在哪里

- `apps/user-app/src/pages/`：页面本体，平台差异优先放 `*.h5.tsx` / `*.rn.tsx`
- `apps/user-app/src/runtime/`：跨页面运行时、上下文 Provider、路由守卫和全局壳层
- `apps/user-app/src/runtime/shared/`：真正要跨 H5 / RN 共享的运行时脚本与纯逻辑

## 跨端导览骨架放在哪里

`Spec 001.9 - 初始化完成后跨端新手向导` 的阶段 1 已经把导览骨架拆成三层：

- `apps/user-app/src/runtime/shared/user-guide/manifest.ts`
  - 共享导览脚本和导览版本号
- `apps/user-app/src/runtime/shared/user-guide/runtime.ts`
  - 纯状态机，不夹 DOM 或 RN 视图细节
- `apps/user-app/src/runtime/h5-shell/user-guide/platformAdapter.ts`
  - H5 平台适配器边界
- `apps/user-app/src/runtime/rn-shell/user-guide/platformAdapter.ts`
  - RN 平台适配器边界
- `apps/user-app/src/runtime/user-guide.tsx`
  - 全局导览 Provider，对外暴露导览会话、当前步骤和完成写回能力

这里的规则别搞错：

- 导览步骤顺序、文案 key、目标路由只能放共享脚本里
- H5 / RN 只允许在平台适配器里处理定位、浮层、锚点和滚动差异
- 不允许在单个页面里各写一套“如果是 H5 就这样、如果是 RN 就那样”的导览流程

## 阶段 2 / 3 之后新增了什么

现在这套导览已经不只是骨架，前端还补上了下面这些关键接缝：

- `apps/user-app/src/runtime/shared/user-guide/GuideAnchor.*`
  - 页面用这个注册稳定锚点，H5 / RN 各自走平台文件，不再靠脆弱类名查 DOM
- `apps/user-app/src/runtime/shared/user-guide/localState.ts`
  - 负责两类本地状态：
    - 初始化完成后的“一次性自动触发”标记
    - 导览进行中的最近一步 checkpoint，用来刷新恢复
- `apps/user-app/src/runtime/shared/user-guide/GuideOverlay.*`
  - 同一个导入路径，按平台分别落 H5 浮层和 RN 浮层
- `apps/user-app/src/runtime/user-guide.tsx`
  - 现在除了 Provider 以外，还负责：
    - 读取成员导览版本
    - 恢复本地最近一步
    - 判断是否因为刚完成初始化而自动拉起
    - 路由切换、锚点等待、跳过/完成写回

## 页面接入导览时怎么写

- 页面里优先接 `GuideAnchor`
  - H5 / RN 都从 `runtime/index.ts` 或 `runtime/index.rn.ts` 导出拿，不要自己绕到平台目录里拼路径
- 锚点 ID 统一用 `USER_GUIDE_ANCHOR_IDS`
  - 不要在页面里手写裸字符串，不然脚本和页面很快会对不上
- 真正适合精确高亮的区域再做细锚点
  - 例如首页欢迎区、记忆列表、设置页“重新查看导览”入口
- 结构还不稳定或仍是占位页的页面，直接做页面级 anchor
  - 不要为了“精确”硬写一堆 RN 测量代码，把第一版做烂

## 自动触发和恢复口径

- 初始化完成页点击“进入应用”前，前端会先写一个一次性本地标记
- 用户进入应用后，如果当前成员还没完成当前导览版本，就自动启动导览
- 导览过程中会把最近一步写到本地 checkpoint
- 如果用户刷新页面或切到别的页面再回来，Provider 会优先尝试恢复最近一步
- 如果本地恢复失败，或者目标区域没渲染出来，系统会降级到当前页继续说明，不允许整条链路卡死

## 本地调试路径

- 导览共享逻辑单测：
  - `npm.cmd --prefix ./apps/user-app run test:guide-runtime`
- 导览相关类型检查：
  - `npm.cmd --prefix ./apps/user-app run typecheck`
- 如果你要联调“初始化完成后自动触发”：
  - 不要去改 `household.setup_status`
  - 直接看 `member_preferences.user_app_guide_version` 和本地 guide checkpoint / auto-start 标记是否一致
