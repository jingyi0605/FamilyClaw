# Installation Overview

## What this page solves

- It helps readers choose the right installation or deployment path.

## Path selection

- Fastest first run: [Docker Installation](./docker-installation.md)
- Source-level development: [Source Installation](./source-installation.md)
- NAS deployment: [NAS Deployment](./nas-deployment.md)
- Linux server rollout: [Ubuntu Deployment](./ubuntu-deployment.md)
- Windows-based validation: [Windows Deployment](./windows-deployment.md)

## Selection rules

- If Docker is enough, do not force source installation first.
- Use source installation when development or deep debugging is the real goal.
- NAS, Ubuntu, and Windows are environment differences, not product differences.

## Completion standard

- The reader knows which deployment page to open next.
