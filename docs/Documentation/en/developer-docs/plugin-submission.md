# Plugin Submission

## Minimum requirements before submission

1. The plugin directory and `manifest.json` follow the rules.
2. Entry paths map to real code.
3. Types, permissions, and risk level are declared clearly.
4. A readable README exists.
5. Validation notes and dependency files are ready.

## Minimum submission package

- Plugin source repository URL
- Plugin documentation URL
- Registry entry
- Pull request description

## What reviewers will check

- Registry entry vs `manifest.json`
- Repository accessibility
- README clarity
- Risk level honesty

## Completion standard

- Third-party authors know how to move from a finished plugin to a formal submission.
