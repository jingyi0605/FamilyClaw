# Docs Overview

## What this page solves

- It helps a new reader find the right section without scanning the whole site.
- It locks the public documentation into five top-level sections.

## Current top-level sections

- `Getting Started`: understand the product first, then choose your next path.
- `Installation`: Docker, source, NAS, Ubuntu, and Windows deployment paths.
- `User Guide`: dashboard, households, conversations, memory, and settings.
- `Developer Docs`: environment setup, backend work, and plugin development.
- `Community`: official website and discussion channels.

## Recommended reading paths

### I am new to FamilyClaw

1. Start with [Product Overview](./product-overview.md).
2. Continue with [Quick Start](./quick-start.md).
3. Then move to installation, usage, or development based on your role.

### I need the English docs structure

1. Stay inside the `/en/` tree.
2. Keep the Chinese root structure unchanged.
3. Add content page by page instead of inventing a second architecture.

## Completion standard

- A reader can find the right entry point in under 30 seconds.
