# 文档图片与排版约定

## 这页要解决什么

- 统一文档里的图片放置方式、插入方式和排版方式
- 避免不同作者各写各的，最后把页面撑坏

## 图片放哪里

### 单页专属图片

- 放在当前文档同目录的 `assets/`
- 命名要能看懂，不要用 `1.png`、`截图(7).png`

### 多页共用图片

- 放在 `docs/Documentation/.vitepress/public/`
- 适合 logo、全站共用示意图、统一流程图

## 图片格式建议

- 优先 `webp`
- 需要保真时用压缩后的 `png`
- 不建议把大体积原始截图直接扔进仓库

## 插图方式

### 普通插图

```md
![页面名称](./assets/example.webp)
```

### 需要控宽度、标题和统一边框时

```md
<DocImage
  src="./assets/example.webp"
  alt="页面名称"
  caption="这里写图注"
  maxWidth="960px"
/>
```

## 排版原则

- 一张图只表达一个重点
- 图放在对应段落后，不要堆到文末
- 超宽截图必须收窄，不准裸贴原图
- 连续步骤图超过 3 张时，优先改成小图加文字，不要刷屏

## 推荐做法

- 先裁掉无关区域，再放图
- 需要强调局部时，优先单独截局部，不要整页瞎框
- 需要写图注时，用 `DocImage`，不要手工拼乱七八糟的 HTML

## 不推荐做法

- 把聊天截图直接贴进正式文档
- 用超大图片代替文字说明
- 同一页混用 3 套图片样式

## 这页完成标准

- 任何人都知道图片该放哪、怎么插、怎么控宽度
