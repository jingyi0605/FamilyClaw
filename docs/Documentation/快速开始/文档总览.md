# 文档总览

## 这页要解决什么

- 第一次进站的人 30 秒内知道先看哪几页
- 把全站固定收口成 5 个一级目录，后续新增内容只能落在这几个桶里

## 当前一级目录

- `快速开始`：认识产品、明确入口，再决定是体验还是开发
- `安装部署`：Docker（推荐）、源码、本地/NAS/Ubuntu/Windows 的安装与部署
- `使用指南`：围绕仪表盘、家庭、对话、记忆、设置讲清楚怎么用
- `开发文档`：开发环境、后端与插件开发的规则、接口入口
- `沟通交流`：官网、QQ群、微信群、Discord 入口

## 建议阅读路径

### 我第一次接触 FamilyClaw

1. 先看 [产品概览](./产品概览.md) 了解做什么。
2. 再看 [快速启动](./快速启动.md) 选体验路径。
3. 需要跑起来：直接跳到 [安装部署](../安装部署/概览.md) 选环境。
4. 想知道怎么用：去 [使用指南](../使用指南/仪表盘.md)。
5. 要开发或二开：去 [开发文档](../开发文档/环境准备.md)。

### 我要给用户发安装文档

1. 先看 [安装部署概览](../安装部署/概览.md) 选 Docker / 源码 / NAS / Ubuntu / Windows。
2. 按环境打开对应页面，直接把链接发给用户。

### 我要补英文文档

1. 打开 `/en/` 首页。
2. 按同样栏目找到对应英文页；不要调整中文根路径结构。

## 完成标准

- 新读者 30 秒内找到入口并知道下一步点哪页
- 新增内容必须落在 5 个一级目录之一，不再横向加新栏
