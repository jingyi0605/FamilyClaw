# 文档总览

如果你是第一次来到这里，不需要先把整套文档看完。

先根据你现在要做的事，选一条最适合自己的路径往下走就够了。

如果你想先知道项目现在做到哪一步、接下来在补什么、后面准备做什么，直接去看 [路线图](./路线图.md)。

## 第一次来，建议这样看

1. 先看 [产品概览](./产品概览.md)，知道 FamilyClaw 能帮你做什么。
2. 再看 [路线图](./路线图.md)，快速判断哪些能力已经能用，哪些还在开发中。
3. 然后看 [快速启动](./快速启动.md)，判断你现在更适合先体验、先安装，还是继续深入使用。
4. 如果你已经很明确自己的目标，可以直接跳到下面对应的入口。

## 按你的目标直接进入

### 我想先把系统装起来

- 从 [安装部署概览](../安装部署/概览.md) 开始。
- 里面会按 Docker、源码、NAS、Ubuntu、Windows 等常见方式带你进入对应步骤。

### 我已经装好了，想知道怎么用

- 直接进入 [使用指南](../使用指南/仪表盘.md)。
- 这里会从实际页面和操作流程出发，带你继续熟悉日常使用。

### 我想先了解产品，再决定下一步

- 先看 [产品概览](./产品概览.md)。
- 看完后如果你想马上开始，再去 [快速启动](./快速启动.md)。

### 我想先看项目做到哪一步

- 直接看 [路线图](./路线图.md)。
- 这里会把已经实现、正在开发、计划实现的能力分开列清楚，避免你拿历史讨论当现状。

### 我要开发、接入或二次修改

- 直接看 [开发文档](../开发文档/环境准备.md)。
- 这里适合开发环境准备、接口接入、插件开发和联调排查。

### 我只是想找官网或社群

- 去 `沟通交流` 目录查看官网和社群入口。

## 这些文档分别适合什么时候看

- `快速开始`：适合第一次接触 FamilyClaw，先建立整体认识。
- `路线图`：适合先确认哪些能力已经落地，哪些还在开发，哪些只是下一阶段计划。
- `安装部署`：适合目标明确，就是要把系统跑起来。
- `使用指南`：适合已经登录进入系统，准备开始正式使用。
- `开发文档`：适合开发、联调、写插件或排查实现问题。
- `沟通交流`：适合想继续关注更新、加入社群或获取外部入口。
