# 路线图

这页只做一件事：把 FamilyClaw 截至 2026-03-22 的现状说清楚。

别把路线图当宣传页。这里不写空话，直接按当前真实状态分成“已完成 / 正在开发 / 计划实现”三类。

## 怎么看这页

| 状态 | 判断标准 | 你该怎么理解 |
| --- | --- | --- |
| 已完成 | 页面、链路或能力已经真实可用，或者已经作为正式能力交付 | 现在就可以按现有系统和文档去用，不要再挂成“进行中” |
| 正在开发 | 已经确定要做，而且目前还在持续实现、联调或优化 | 方向已经锁定，但还没有完全收口 |
| 计划实现 | 方向已经明确，但还没进入当前交付主线 | 先知道有这件事，但别按“已经支持”理解 |

## 该怎么用这页

- 如果你现在就要部署和使用，优先看“已实现”里的文档入口。
- 如果你要判断某项能力能不能对外承诺，先看它是在“已实现”还是只是“正在开发 / 计划实现”。
- 如果你准备参与开发，最该先看的不是这里，而是对应的正式文档和 Spec。

## 结论

FamilyClaw 现在已经不是空壳了，安装、首页、家庭、对话、记忆、设置、插件、设备接入和多端基础能力都已经完成，不该继续挂在“开发中”里假装半成品。

现在真正还在推进的重点，只保留长期记忆召回、人格化 Agent、声纹优化、多平台通讯通道、移动客户端，以及基于小米摄像机的人脸识别与视觉分析。路线图的作用，就是把这条边界写明白。
