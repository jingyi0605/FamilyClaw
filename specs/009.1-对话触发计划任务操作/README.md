# 009.1 - 对话触发计划任务操作

状态：Draft

## 这份 Spec 解决什么问题

`009-计划任务系统` 已经把计划任务底座、前后端接口、规则触发、Agent 提醒和“独立草稿接口”做出来了，但还差最后一段最关键的体验：

- 用户在正常聊天里说一句“每天晚上九点提醒我吃药”
- 系统能识别这是计划任务，不是普通提醒
- 系统能在聊天主链里给出确认卡片
- 用户确认后正式创建任务

现在项目里已经有一条“独立接口式草稿链路”，但还没接进现有 `conversation proposal pipeline`。这份 Spec 就是把这块补齐。

## 目录

- `requirements.md`：为什么要做、要做到什么程度
- `design.md`：怎么把计划任务接进现有对话主链
- `tasks.md`：具体实现顺序和验收方式

## 和 `009` 的关系

- `009` 负责整个计划任务系统
- `009.1` 只负责“对话里原生触发计划任务操作”这一段补齐
- 换句话说，这不是再造第二套计划任务系统，而是把已有底座接进聊天主链
