# 20260314-家庭地区接口样例

这份文档只做一件事：把这次 Spec 里最关键的地区接口，用前后端和插件都能直接看懂的方式写出来。

先说明边界：

- 这里只给样例，不替代正式 OpenAPI
- 第一版只展示中国大陆区县场景
- 这里只覆盖地区目录、家庭资料、插件地区上下文，不展开天气业务本身

## 6. 插件读取家庭地区上下文

### 6.1 读取成功

这里是服务调用样例，不是用户直接访问的 HTTP 接口。

调用：

```json
{
  "tool": "region.resolve_household_context",
  "input": {
    "household_id": "household-001"
  }
}
```

返回：

```json
{
  "status": "configured",
  "provider_code": "builtin.cn-mainland",
  "country_code": "CN",
  "region_code": "110108",
  "admin_level": "district",
  "province": {
    "code": "110000",
    "name": "北京市"
  },
  "city": {
    "code": "110100",
    "name": "北京市"
  },
  "district": {

    "name": "海淀区"
```

## 7. 前后端和插件要对齐的几条死规矩

- 新入口优先提交 `region_selection`，不要再把 `city` 当正式地区来源
- `city` 只是兼容展示字段，不再承担地区识别责任
- 正式家庭绑定必须落到区县级，市级和省级都不算完成
- 插件读取地区时只走 `region.resolve_household_context`，不要自己拼 `households.city`
- 如果以后接海外地区，继续复用 `provider_code + country_code + region_code` 这一套，不要再加一套平行模型
