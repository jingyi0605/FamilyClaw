# Spec 004.4 - 用户前端插件管理与插件市场

状态：Draft

## 这份 Spec 解决什么问题

后端插件系统已经有了，但前端如果只做“列表 + 开关”，那用户还是看不懂插件系统到底能做什么，也配不起来。

这份 Spec 负责把用户真正看得见、配得上的前端入口收口出来。

## 这次要做到什么

- 让用户看到已安装插件、状态、风险和来源
- 让用户能进入插件详情，而不是只看一排卡片
- 让插件配置入口统一，不再每个插件各开一套特例页面
- 让插件市场能区分内置、官方、第三方来源
- 让插件任务状态对用户可见，不再假装“点一下就同步完成”

## 现在要补上的新事实

前端插件管理已经不只是“开关列表”。

这轮之后至少要承认两件事：

1. `PluginDetailDrawer` 已经是统一插件配置入口的一部分
2. 正式配置协议和动态表单细节由 `../004.2.3-插件配置协议与动态表单/` 承接

也就是说：

- 这份 Spec 讲“前端插件管理怎么消费”
- `004.2.3` 讲“配置协议本身长什么样”

## 这次明确不做什么

- 不做一键远程安装任意仓库代码
- 不做复杂推荐系统
- 不在这里复制配置协议字段表

## 关于旧插件

没有配置协议的旧插件，前端也必须继续能展示、启停、执行。

只是：

- 有配置协议的插件，会多出统一配置入口
- 没配置协议的插件，仍然保留基础管理能力

这条兼容性不能破。
