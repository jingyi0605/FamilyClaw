# 002.3 分层长期记忆与召回机制

状态：Draft

## 这份 Spec 解决什么问题

当前项目已经有 `memory_cards`、事件入库、记忆候选和基础权限控制，但还没有真正做成一套长期记忆系统。现在的主要问题很直接：

- 记忆只有一层，缺少会话记忆、情节记忆和外部知识层。
- 检索还是规则打分，缺少 PostgreSQL 原生的关键词和向量召回。
- `free_chat` 只吃热摘要，没有把真正命中的记忆结果分组注入主链路。
- 会话结束后没有稳定沉淀，最近说过什么、刚确认过什么，很快就丢了。

这份 Spec 的目标是把长期记忆收口成一套可以演进的分层方案：

- `L0` 当前轮上下文：最近对话 + 设备上下文
- `L1` 会话记忆：滚动摘要、未完成话题、最近确认事项
- `L2` 情节记忆：带时间戳的事件/会话摘要
- `L3` 语义记忆：稳定事实、偏好、关系
- `L4` 外部知识：插件原始记录、说明文档、规则文档

最终形态不是把 Markdown 文件硬塞进系统，而是：

- PostgreSQL 结构化数据做真相源
- `tsvector + GIN` 和 `pgvector` 做召回投影
- `session summary` 做中间层
- 统一召回结果真正接进对话主链路

## 当前结论

- 第一阶段先闭环 `memory_cards` 的关键词/向量召回和 `free_chat` 注入。
- 第二阶段补 `conversation_session_summaries`，把会话层记忆补齐。
- 第三阶段再把情节记忆和外部知识纳入统一召回，不在第一版里过度设计。
