# Spec 001.3.1 - 家庭地区坐标扩展

状态：Draft

## 这份 Spec 解决什么问题

现在家庭地区体系只有“绑定到哪个地区节点”，但没有稳定坐标。天气这种能力真正依赖的是经纬度，不是城市名字。

继续拿 `city` 文本或者地区名称去猜，会很快变烂：

- 同名城市、区县会匹配错
- 海外挂件地区包很难复用同一套查询逻辑
- 家庭真实位置和行政地区代表点会被混成一个概念

这份 Spec 的目标很直接：给现有地区体系补上坐标能力，而且把两类坐标分开。

- 地区代表坐标：由地区插件包提供，表示这个地区节点的默认查询点
- 家庭精确坐标：由用户主动确认后保存，用来覆盖默认代表点

## 这次要做什么

- 让内置地区和外挂地区插件包都可以输出区县级代表坐标
- 给家庭增加可选的精确坐标字段，但不改变现有地区绑定关系
- 提供统一的“坐标解析结果”，给天气等插件直接消费
- 明确坐标来源、精度、更新时间语义，避免以后越写越乱

## 这次明确不做什么

- 不做自动后台持续定位
- 不做浏览器定位一拿到就自动覆盖家庭位置
- 不做地图选点页面和轨迹能力
- 不做天气业务本身，这份 Spec 只负责把坐标基础补齐

## 和上游 Spec 的关系

- 继承 `../001.3-家庭地区模型与插件接口/` 的地区模型和 provider 扩展方向
- 为 `../004.5.1-官方天气插件/` 提供稳定的经纬度输入
