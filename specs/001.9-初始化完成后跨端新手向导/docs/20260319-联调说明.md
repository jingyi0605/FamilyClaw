## 先看哪几个状态

联调时先别盯 UI，先把状态源看清楚。

- 家庭初始化状态：
  - 仍然看 `household.setup_status`
  - 它只负责初始化准入，不再承载导览完成状态
- 成员导览状态：
  - 看 `member_preferences.user_app_guide_version`
  - 它表示当前成员已经处理到哪一版导览
- 本地一次性触发标记：
  - key：`familyclaw.userAppGuide.pendingAutoStart`
  - 用途：初始化完成页进入应用前写入，首进应用时消费并删除
- 本地恢复 checkpoint：
  - key：`familyclaw.userAppGuide.session`
  - 用途：导览进行中保存最近一步，刷新或切页回来时尝试恢复

## 自动触发规则

只有下面两件事同时成立，才会自动拉起导览：

1. 初始化完成页刚刚写入了一次性自动触发标记
2. 当前成员的 `user_app_guide_version` 小于当前导览版本号

任何一个条件不满足，都不会自动触发。

## 恢复和降级规则

- 导览进行中，每次步骤变化都会写本地 checkpoint
- 页面刷新后，Provider 会优先尝试恢复最近一步
- 如果 checkpoint 版本过期、成员已经完成当前导览、或者恢复目标步骤无效，会直接清理 checkpoint，不会卡死
- 如果目标 anchor 等待超时，导览会退回页面级说明，继续当前步骤，不会因为单个 anchor 缺失把整条链路拖死

## 手工联调建议顺序

### H5

1. 用新成员完成初始化流程，进入应用
2. 确认首页首进自动拉起导览
3. 顺着导览依次经过首页、家庭、助手、记忆、设置
4. 在任意一步刷新页面，确认能恢复到最近一步
5. 在任意一步点击跳过，确认：
   - 导览关闭
   - `user_app_guide_version` 已写回当前版本
   - 本地 auto-start 标记和 checkpoint 已清理
6. 进入设置页，点击“重新查看导览”，确认能重新拉起
7. 人为让某个 anchor 不可用，确认页面级说明顶上来，而不是整条流程中断

### RN

1. 用未完成当前导览版本的成员进入应用
2. 确认首页能拉起同一套共享脚本
3. 顺着导览走到设置页，确认占位页和页面级 anchor 也能继续链路
4. 在步骤中切页或返回，确认恢复逻辑正常
5. 从设置页手动重开，确认按钮可用
6. 确认某个精确 anchor 缺失时，仍会回退到页面级说明
