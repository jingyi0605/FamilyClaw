# 补充文档目录

`docs/` 用来放这份 Spec 后续实现和验收时会直接用到的补充材料。

建议优先补这些文档：

- `issue-form.md`
  - 写 Issue Form 字段说明、示例和填写注意事项
- `validation-contract.md`
  - 写 Issue 解析对象、校验结果对象和错误回写格式
- `validation-rules.md`
  - 写插件仓库、`manifest.json`、README、版本信息的自动校验规则
- `github-actions.md`
  - 写 GitHub Actions 工作流触发条件、权限和 PR 更新规则
- `review-checklist.md`
  - 写维护者审核机器人 PR 时该看什么
- `acceptance-checklist.md`
  - 写端到端验收清单

如果当前还没开始补这些文档，先保留本文件就够了。
