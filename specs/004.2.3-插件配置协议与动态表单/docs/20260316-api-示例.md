# 插件配置 API 示例

这几个接口就是第一版正式入口。别再从别的接口偷读偷写配置。

## 1. 获取插件可配置作用域

请求：

```http
GET /api/v1/ai-config/{household_id}/plugins/{plugin_id}/config-scopes
```

响应示例：

```json
{
  "plugin_id": "demo-plugin-config",
  "items": [
    {
      "scope_type": "plugin",
      "title": "演示插件配置",
      "description": "当前家庭下只有一份插件级配置。",
      "instances": [
        {
          "scope_key": "default",
          "label": "默认配置",
          "description": "当前插件在这个家庭下只有一份插件级配置。",
          "configured": false
        }
      ]
    }
  ]
}
```

## 2. 读取某个作用域的配置表单

请求：

```http
GET /api/v1/ai-config/{household_id}/plugins/{plugin_id}/config?scope_type=plugin&scope_key=default
```

响应示例：

```json
{
  "plugin_id": "demo-plugin-config",
  "config_spec": {
    "scope_type": "plugin",
    "title": "演示插件配置",
    "schema_version": 1
  },
  "view": {
    "scope_type": "plugin",
    "scope_key": "default",
    "schema_version": 1,
    "state": "configured",
    "values": {
      "base_url": "https://example.com/api",
      "mode": "strict",
      "metadata": {
        "region": "cn"
      }
    },
    "secret_fields": {
      "api_key": {
        "has_value": true,
        "masked": "******"
      }
    },
    "field_errors": {}
  }
}
```

## 3. 保存配置

请求：

```http
PUT /api/v1/ai-config/{household_id}/plugins/{plugin_id}/config
Content-Type: application/json
```

```json
{
  "scope_type": "plugin",
  "scope_key": "default",
  "values": {
    "base_url": "https://example.com/api",
    "mode": "strict",
    "api_key": "secret-token-001"
  }
}
```

## 4. secret 保留和清空语义

保留旧值：

- 请求里不带这个 secret 字段
- 后端继续保留库里的旧值

清空旧值：

```json
{
  "scope_type": "channel_account",
  "scope_key": "channel-account-001",
  "values": {},
  "clear_secret_fields": ["webhook_secret"]
}
```

## 5. 保存失败时长什么样

如果字段校验不过，接口直接返回错误，别假装保存成功。

响应示例：

```json
{
  "detail": {
    "detail": "插件配置校验失败。",
    "error_code": "plugin_config_validation_failed",
    "field_errors": {
      "api_key": "API Key 不能为空"
    }
  }
}
```

## 6. 兼容旧 channel 配置的规则

- `channel_account` 读取时，会先兼容旧的 `channel_plugin_accounts.config_json`
- 一旦通过新接口保存，就会同时写 `plugin_config_instances` 和旧 `config_json`
- 这样做不是优雅，是为了不破坏现有通道运行链路
