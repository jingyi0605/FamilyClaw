# 插件 manifest 配置协议示例

这份示例不是拿来展示“协议多灵活”，而是给后面接插件的人直接照着改。

第一原则只有一条：

- 字段定义只保留一份来源，放在插件 `manifest.json`

## 什么时候该写 `plugin` 作用域

插件只有一份全局配置，而且这份配置跟具体通道账号无关时，用 `plugin`。

例子：

- 一个同步插件自己的 API 地址
- 一个动作插件自己的默认参数

## 什么时候该写 `channel_account` 作用域

每个通道账号各自一份配置时，用 `channel_account`。

例子：

- Telegram Bot Token
- Discord Bot Token
- 企业微信机器人 Webhook Key

## 最小可用示例

```json
{
  "id": "demo-plugin-config",
  "name": "演示插件配置",
  "version": "0.1.0",
  "types": ["connector"],
  "permissions": ["health.read"],
  "risk_level": "low",
  "triggers": ["manual"],
  "entrypoints": {
    "connector": "plugin.connector.sync"
  },
  "config_specs": [
    {
      "scope_type": "plugin",
      "title": "演示插件配置",
      "description": "这是插件级配置，当前家庭下只有一份。",
      "schema_version": 1,
      "config_schema": {
        "fields": [
          {
            "key": "base_url",
            "label": "基础地址",
            "type": "string",
            "required": true
          },
          {
            "key": "mode",
            "label": "模式",
            "type": "enum",
            "required": true,
            "default": "strict",
            "enum_options": [
              { "value": "strict", "label": "严格" },
              { "value": "loose", "label": "宽松" }
            ]
          },
          {
            "key": "api_key",
            "label": "API Key",
            "type": "secret",
            "required": true
          },
          {
            "key": "metadata",
            "label": "额外配置",
            "type": "json",
            "required": false
          }
        ]
      },
      "ui_schema": {
        "sections": [
          {
            "id": "basic",
            "title": "基础参数",
            "fields": ["base_url", "mode", "api_key", "metadata"]
          }
        ],
        "widgets": {
          "mode": { "widget": "select" },
          "api_key": { "widget": "password" },
          "metadata": { "widget": "json_editor" }
        }
      }
    }
  ]
}
```

## 第一版别做什么

- 别把它做成完整 JSON Schema 工程
- 别塞远程异步选项源
- 别塞脚本表达式
- 别在前端页面里再维护一份同样的字段常量

## secret 字段记住两条

- 读取时永远只给 `has_value` 和掩码，不回明显文
- 保存时没提交这个字段，就表示保留原值；要清空就走 `clear_secret_fields`
