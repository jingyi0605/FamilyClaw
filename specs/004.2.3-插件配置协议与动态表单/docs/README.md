# 文档目录说明 - 插件配置协议与动态表单

状态：Draft

这个目录不是拿来堆空文档的，只放后续实现时真会用到的补充材料。

建议后续文档按下面几类补：

- `api-示例.md`
  - 放配置读取、保存、字段报错、secret 清空这些请求与响应示例
- `manifest-示例.md`
  - 放插件 `config_schema` / `ui_schema` 示例，给后续插件直接照着写
- `channel-迁移说明.md`
  - 放 `channel` 从硬编码字段迁到统一协议的改造步骤、兼容点和回归点
- `字段字典.md`
  - 放第一版字段类型、widget、显示条件、secret 语义说明
- `联调清单.md`
  - 放前后端联调时要核对的项目，比如默认值、字段错误、secret 保留、clear 行为

补文档时记住三条死规矩：

1. 先写人话，别写黑话。
2. 先写“别人接手时最容易踩坑的点”，别先堆概念。
3. 示例必须贴近仓库现状，别写一份谁都不会照着用的假样例。
