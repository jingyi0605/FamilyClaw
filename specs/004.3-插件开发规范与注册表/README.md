# Spec 004.3 - 插件开发规范与注册表

状态：Draft

## 现在这份 Spec 只负责什么

这份 Spec 继续负责两件事：

- 插件开发规范
- 插件注册表

但插件系统的正式能力边界、类型体系、记忆槽位和迁移策略，统一以这些文档为准：

- `docs/开发设计规范/20260318-插件能力与接口规范-v1.md`
- `docs/开发设计规范/20260318-现有插件系统迁移路线图.md`
- `docs/开发设计规范/20260318-记忆中心内核与插件边界规范.md`

## 这轮必须按什么理解

- 普通状态型插件统一归到 `integration`
- 不再把 `connector`、`memory-ingestor` 当正式主语义
- 记忆能力不是普通插件类型，而是独占槽位
- 正式开发者手册以 `docs/开发者文档/插件开发/` 为主入口

## 这份 Spec 不再做什么

- 不再复制维护第二套 manifest 正文
- 不再单独定义旧 `connector` / `memory-ingestor` 规则
- 不再把主题、AI 供应商、记忆能力当成“像插件的例外”
