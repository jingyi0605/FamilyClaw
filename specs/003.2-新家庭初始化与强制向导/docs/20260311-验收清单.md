# 003.2 验收清单

## 验收目标

确认 `003.2` 不再只是几块散功能，而是一条真的能从“新家庭创建”走到“主链路放行”的正式路径。

## 一、读模型

- [ ] 调 `GET /api/v1/households/{household_id}/setup-status` 能拿到统一结构
- [ ] 返回中包含 `status`
- [ ] 返回中包含 `current_step`
- [ ] 返回中包含 `missing_requirements`
- [ ] 返回中包含 `is_required`

## 二、新家庭强制链路

- [ ] 新建家庭后访问 `/`
- [ ] 确认被重定向到 `/setup`
- [ ] 向导第一步能保存家庭资料
- [ ] 向导第二步能创建首位成员
- [ ] 向导第三步能完成最小供应商配置
- [ ] 向导第四步能创建首个 `butler`
- [ ] 四步完成后再访问 `/`
- [ ] 四步完成后再访问 `/conversation`
- [ ] 四步完成后再访问 `/settings/ai`
- [ ] 上述页面都能正常放行

## 三、历史家庭兼容

### 3.1 已可用历史家庭

- [ ] 准备一个历史家庭
- [ ] 该家庭已有 `qa_generation` 路由
- [ ] 该家庭已有激活的 `butler`
- [ ] 即使缺 `city`，访问 `/` 也不会被锁死
- [ ] 访问 `/settings/ai` 可以看到补录入口

### 3.2 不可用历史家庭

- [ ] 准备一个历史家庭
- [ ] 删除或不配置 `qa_generation` 路由
- [ ] 或者没有激活的 `butler`
- [ ] 访问 `/conversation` 会被带到 `/setup`
- [ ] `/setup` 显示的是缺失步骤，不是从头乱来

## 四、恢复路径

- [ ] 向导做到一半时刷新页面
- [ ] 刷新后仍能根据读模型恢复到当前步骤
- [ ] 已完成步骤不会因为刷新丢失
- [ ] 历史家庭手动进入 `/setup` 时能看到当前缺口

## 五、构建验证

- [ ] `cd apps/api-server && python -m compileall app`
- [ ] `cd apps/user-web && npm.cmd run build`

## 六、这轮明确不验什么

- [ ] 不验完整用户端 AI 配置中心
- [ ] 不验复杂供应商高级参数
- [ ] 不验首个管家的对话式创建
- [ ] 不验 `admin-web` 的改造
