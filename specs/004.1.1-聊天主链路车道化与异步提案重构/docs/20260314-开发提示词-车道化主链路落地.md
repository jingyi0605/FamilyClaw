# 开发提示词 - 车道化主链路落地

日期：2026-03-14

## 用途

这份文档是给后续新会话直接开工用的，不是讲概念，是让开发者少走弯路。

使用方式：

- 新开会话时，直接复制下面的提示词
- 不要让 AI 自己重新猜架构方向
- 直接让它按当前 Spec 约束往下做

## 提示词 1：从 Spec 直接开始开发

```text
请按 `specs/004.1.1-聊天主链路车道化与异步提案重构` 开始实现，不要在 `004.1` 上继续打补丁。

必须先阅读这些文件：
- `specs/004.1.1-聊天主链路车道化与异步提案重构/README.md`
- `specs/004.1.1-聊天主链路车道化与异步提案重构/requirements.md`
- `specs/004.1.1-聊天主链路车道化与异步提案重构/design.md`
- `specs/004.1.1-聊天主链路车道化与异步提案重构/tasks.md`
- `specs/004.1.1-聊天主链路车道化与异步提案重构/docs/20260314-004.1与004.1.1迁移边界说明.md`

严格遵守下面这些约束：

1. `004.1` 是当前生产基线，不允许因为新链路开发破坏当前正常聊天主回复、流式返回、会话持久化和旧确认入口。
2. `004.1.1` 只做渐进式迁移，先加新路，不先拆旧路。
3. 第二层轻语义路由方案已经定死为 `Embedding + Capability Descriptor Retrieval`，不要改成关键词主路由，不要改成小模型分类器，不要额外再打一轮大模型正文理解。
4. 默认内置 Embedding 模型已经定死为 `BAAI/bge-small-zh-v1.5`，同时要支持用户自定义 Embedding provider。
5. analyzer 是逻辑组件，不等于一次独立 LLM 调用。默认必须先 gate，再做一次统一 proposal batch extraction，不允许 memory/config/reminder 各打一轮大模型。
6. 所有实现优先遵守 `tasks.md` 的阶段顺序，尤其要先做影子模式、灰度开关和回滚路径，不要一上来就接管主路由。
7. 每完成一个 task，要及时回写 `tasks.md` 对应任务状态。

这次请先完成 `tasks.md` 里的 `1.3 落轻语义路由的 descriptor 和 Embedding provider`。

开始前请先输出：
- 你对这个 task 的理解
- 你准备修改哪些文件
- 你准备如何保证不影响 `004.1` 当前基线

然后再开始实施。
```

## 提示词 2：只做 Embedding 基础设施

```text
请只实现 `specs/004.1.1-聊天主链路车道化与异步提案重构/tasks.md` 中的 `1.3 落轻语义路由的 descriptor 和 Embedding provider`，不要顺手改其它阶段任务。

必须遵守：

- 默认内置 Embedding 模型固定为 `BAAI/bge-small-zh-v1.5`
- provider 模块落点遵守 `design.md` 中 4.3.6 的建议代码结构草图
- 配置项命名遵守 `design.md` 中已经写死的字段名
- 外部 provider 不可用时，默认允许回退到内置 provider
- descriptor embedding 需要支持预计算和缓存
- 当前主聊天链路默认不接管，只允许以开关方式影子接入

这次实施后请至少给出：

1. 新增了哪些模块和配置项
2. 如何加载默认内置模型
3. 如何切换到用户自定义 provider
4. 如何保证关闭开关后完全不影响 `004.1` 基线
5. 运行或测试了哪些验证
```

## 提示词 3：只做语义路由，不做主路由接管

```text
请实现 `Embedding + Capability Descriptor Retrieval` 的第二层轻语义路由，但暂时不要让它接管顶层主路由。

目标：
- 能基于 descriptor 输出 lane scores
- 能基于 descriptor 输出 proposal gate scores
- 先支持影子模式日志输出
- 默认不改变当前主回复行为

必须遵守：

- 第一层仍然是硬信号判断
- 第二层只在第一层未命中时运行
- Top1 lane score `< 0.55` 时保守回到 `free_chat`
- proposal gate score `< 0.62` 时不触发提案提取
- 只允许通过开关启用：
  - `conversation_lane_shadow_enabled`
  - `conversation_embedding_provider_enabled`
  - `conversation_embedding_fallback_to_builtin_enabled`

请先实现最小可运行版本，并补最关键的单测：
- descriptor 命中测试
- 低分回退测试
- provider 故障回退测试
```

## 提示词 4：接手新会话时的固定提醒

```text
开始开发前，请先确认并遵守下面几点，不要自行改方向：

- `004.1` 负责今天还能稳，`004.1.1` 负责明天不会烂
- 迁移策略是影子模式 -> 兼容灰度 -> 主路由灰度 -> 全量接管
- 不允许为了追新架构破坏当前正常聊天链路
- 第二层语义路由唯一方案已经确定，不要重新提备选
- 默认内置 Embedding 模型已经确定，不要重新选型
- 任何实现都要优先考虑可回滚、可灰度、可观测

请先读 Spec，再开始改代码。
```
