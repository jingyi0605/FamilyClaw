# 20260309-家居上下文中心-前端原型说明-v0.1

## 1. 文档目的

说明 `Spec 002` 当前阶段已经落地的前端原型页面，包括页面入口、数据来源、交互方式和已知边界。

## 2. 页面入口

- 前端工程：`apps/admin-web`
- 页面组件：`apps/admin-web/src/pages/ContextCenterPage.tsx`
- 路由：`/context-center`

## 3. 页面组成

当前页面包含四块核心内容：

1. 家庭状态 Hero 区
2. 关键指标仪表盘
3. 成员状态与房间热区面板
4. 上下文配置界面

其中配置界面覆盖：

- 家庭模式
- 隐私模式
- 自动化等级
- 访客模式 / 儿童保护 / 老人关怀 / 语音快路径 / 静音时段
- 成员在家状态、活动状态、所在房间、置信度
- 房间策略与房间级隐私保护

## 4. 数据来源

### 4.1 当前阶段真实接口

页面会并发读取：

- `GET /api/v1/households/{id}`
- `GET /api/v1/members`
- `GET /api/v1/rooms`
- `GET /api/v1/devices`
- `GET /api/v1/audit-logs`

### 4.2 当前阶段本地草稿

由于后端 `context overview / context configs` 还未完成，以下内容暂由浏览器本地持久化承接：

- 家庭模式
- 成员在家状态草稿
- 房间策略草稿
- HA 状态与自动化等级草稿

本地存储键按家庭维度区分，避免多个家庭相互污染。

## 5. 交互策略

- 页面采用 `Promise.allSettled` 拉取数据，单个接口失败不会导致整页白屏
- 配置修改后支持“保存演示配置”
- 刷新页面后可恢复上次本地草稿
- 切换家庭后会切换到对应家庭的上下文草稿

## 6. 已知限制

1. 当前成员状态不是后端真实推理结果，而是演示配置
2. 房间占用由页面根据成员所在房间草稿推导
3. `Home Assistant` 健康状态当前为前端草稿，不是实时后端探活
4. 页面尚未对接 `context overview / context configs / device-actions` 的正式接口

## 7. 后续替换原则

后续接入后端真实接口时，必须遵循：

1. 保留当前 UI 信息架构，不重做页面结构
2. 用后端真实返回替换本地草稿来源
3. 保留本地草稿作为开发或演示兜底模式
4. 将接口联调说明补充到 `specs/002-家居接入与上下文中心/docs/` 下
