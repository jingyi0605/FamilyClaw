# 20260317-迁移验收清单

## 这份清单用来验什么

这份清单只验一件事：`005.3.2` 迁移完成后，本地默认是否已经可以只启动 `api-server`，并且仓库里只剩 `embedded / disabled` 两种有效状态。

## 先看什么

- `apps/api-server/.env.example`
- `apps/start-api-server.sh`
- `apps/api-server/tests/test_voice_runtime_client.py`
- `apps/api-server/tests/test_voice_pipeline.py`
- `apps/api-server/tests/test_voiceprint_async_service.py`
- `apps/api-server/tests/test_voice_realtime_ws.py`

## 验收项 1：本地默认模式是不是 `embedded`

通过标准：

- `apps/api-server/.env.example` 默认写的是 `FAMILYCLAW_VOICE_RUNTIME_MODE=embedded`
- 注释里明确写清楚“本地默认只需要启动 `api-server`”
- `apps/start-api-server.sh` 启动日志会提示当前只需启动 `api-server`

## 验收项 2：`embedded` 模式是不是能自己产出 transcript 和 artifact

通过标准：

- `voice_runtime_client` 仍然是统一入口
- `embedded` 模式下，`audio.append` 只做轻量缓存和统计
- `audio.commit` 时由 blocking helper 在线程池里做落盘、hash 和 artifact 生成
- `voice_pipeline` 在 `embedded` 模式下能继续走 transcript + artifact 主链

建议验证命令：

```bash
cd apps/api-server
.venv\Scripts\python.exe -m unittest tests.test_voice_runtime_client tests.test_voice_pipeline
```

## 验收项 3：独立 `voice-runtime` 是不是已经彻底删除

通过标准：

- `apps/api-server/.env.example` 不再保留 `FAMILYCLAW_VOICE_RUNTIME_BASE_URL` 和 `FAMILYCLAW_VOICE_RUNTIME_API_KEY`
- `apps/start-voice-runtime.sh` 已删除
- `apps/voice-runtime/` 已删除
- `voice_runtime_client` 不再包含 remote backend 选择分支

## 验收项 4：声纹重活是不是已经离开主事件循环

通过标准：

- 建档和普通识别都经过 `app.modules.voiceprint.async_service`
- facade 内部统一走 `run_blocking_db(...)`
- 线程池里使用新的 `SessionLocal`，不复用当前请求或 WebSocket 持有的 Session
- 有回归测试证明慢声纹任务不会拖死同一 worker 的其他 HTTP / WebSocket 请求

建议验证命令：

```bash
cd apps/api-server
.venv\Scripts\python.exe -m unittest tests.test_voiceprint_async_service tests.test_voice_identity tests.test_voice_pipeline tests.test_voice_realtime_ws
```

## 验收项 5：旧 spec 有没有一起回写

通过标准：

- `specs/005.3-小爱声纹采集与身份识别/README.md` 不再暗示本地必须单独起 `voice-runtime`
- `specs/005.3-小爱声纹采集与身份识别/requirements.md` 的运行时表述改成 runtime backend
- `specs/005.3-小爱声纹采集与身份识别/design.md` 的拓扑图和模块职责与“只保留 embedded/disabled”对齐
- `specs/005.3-小爱声纹采集与身份识别/tasks.md` 顶部有迁移后说明，避免继续把历史实现当当前默认结构
