# 共享边界速记

## 这份速记是干什么的

这不是正式设计文档的替代品，而是给后面接手的人一个一眼能看懂的版本：

- 该共享什么
- 不该共享什么
- 页面迁移时先看哪几层

## 一句话结论

`H5` 和 `RN` 共享的是语义，不共享的是具体样式实现。

## 该共享的东西

- 基础 token：颜色、字号、间距、圆角、阴影
- 语义 token：页面背景、卡片背景、主按钮、危险状态、次级文本
- 组件语义：按钮、卡片、表单项、页头、空状态、区块标题
- 页面布局模式：桌面 H5、移动 H5、RN 这三类场景下的结构规则

## 不该共享的东西

- `scss/css` 文件本身
- `StyleSheet.create(...)` 本身
- `media query`
- `hover`
- 滚动条样式
- `SafeArea`
- 原生 `elevation`
- 浏览器专属 `position: fixed` 细节

## 页面迁移时先看什么

1. 先看共享契约够不够
2. 再看 H5 和 RN 适配层有没有偷跑分叉
3. 最后再改页面

## 页面迁移时别干什么

- 别为了省事把 `H5` 的 CSS 直接翻译成 `RN StyleSheet`
- 别为了追求“跨端统一”把浏览器特性塞进共享层
- 别在页面里继续补新的散装尺寸和颜色
