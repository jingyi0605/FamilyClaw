# 登录页移动端补充验收

## 这次补的是什么

这次不是再开一轮新抽象，而是把当前已经接入 H5 主题体系的登录页补到能在手机浏览器里正常用。

之前的问题很直接：

- 登录卡片最大宽度只有 `420px`，手机上看起来像桌面版缩略图
- 品牌区占得太高，导致真正要点的表单被压到下方
- 输入框和提交按钮高度偏小，触屏点击手感差
- 顶部语言和主题开关在窄屏里太挤

## 实际落地

- 调整 [`index.scss`](/C:/Code/FamilyClaw/apps/user-app/h5-styles/runtime/index.scss) 里的 `.login-*` 样式
- 调整 [`LoginPage.tsx`](/C:/Code/FamilyClaw/apps/user-app/src/runtime/h5-shell/components/LoginPage.tsx) 里的移动端条件渲染
- 停用 [`styles-entry.h5.ts`](/C:/Code/FamilyClaw/apps/user-app/src/pages/login/styles-entry.h5.ts) 对历史 [`pages/login.scss`](/C:/Code/FamilyClaw/apps/user-app/h5-styles/pages/login.scss) 的引入，避免旧的窄卡片规则覆盖当前运行时样式
- 移动端下把登录卡片放大到接近满宽，但保留安全边距
- 压缩品牌区顶部留白，把页面重心拉回到登录表单
- 移动端下直接不渲染品牌说明和 feature 列表，避免手机首屏继续浪费在宣传信息上
- 把 LOGO 图标和 `FamilyClaw` 改成上下排布，并放大到和登录卡片更匹配的体量
- 提高输入框、错误提示和提交按钮的高度与字号
- 收紧顶部控件和下方页脚，避免手机首屏被无效留白吃掉

## 现在怎么验

1. 运行 `npm run dev:user-app:h5`
2. 桌面浏览器切到移动设备模式，确认登录卡片宽度明显放大，没有横向溢出
3. 真机和电脑放在同一局域网，打开 dev server 打印出来的局域网地址
4. 检查输入框、按钮、语言切换和主题切换在触屏宽度下是否还能稳定点按

## 这一步没有做什么

- 没改登录页的交互逻辑和接口调用
- 没把 H5 的 `scss` 直接拿去和 RN 共用
- 没动原生 RN 登录页，因为这次补的是浏览器里的 H5 触屏体验
