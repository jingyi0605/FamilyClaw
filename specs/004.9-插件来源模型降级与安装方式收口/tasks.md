# 004.9 任务清单：插件来源模型降级与安装方式收口

这份任务清单只做一件事：把旧 `official` 模型彻底收口到 `builtin` / `third_party + install_method`，并把 `plugins-dev` 变成可加载的开发源码目录。

---

## 阶段 1：收口正式模型

- [x] 1.1 删除 `official` 作为正式运行时类型
  - 这一步到底做什么：统一后端模型、DTO、枚举和兼容入口，只保留 `builtin`、`third_party`
  - 做完以后能看到什么结果：代码里不再把 `official` 当成当前类型继续分支
  - 这一步依赖什么：无
  - 主要改哪些文件：插件模型、schema、服务层、迁移脚本
  - 这一步明确不做什么：不改开发源码目录行为
  - 怎么验证：搜索正式写路径；相关单测通过

- [x] 1.2 把第三方安装方式收口到 `local` / `marketplace`
  - 这一步到底做什么：把原来的 `manual`、`trusted_level` 等旧口径收口到安装方式
  - 做完以后能看到什么结果：目录生成、安装、升级、卸载都按安装方式走
  - 这一步依赖什么：1.1
  - 主要改哪些文件：安装服务、市场服务、启动同步、清理逻辑
  - 这一步明确不做什么：不新增新的来源类型
  - 怎么验证：本地安装和市场安装相关测试通过

---

## 阶段 2：引入 plugins-dev 开发源码目录

- [x] 2.1 让 `plugins-dev` 参与家庭插件注册表
  - 这一步到底做什么：扫描 `apps/api-server/plugins-dev/`，生成开发版第三方插件记录
  - 做完以后能看到什么结果：开发中的插件不安装也能出现在插件管理页
  - 这一步依赖什么：1.1、1.2
  - 主要改哪些文件：`app/modules/plugin/service.py`
  - 这一步明确不做什么：不写安装 mount
  - 怎么验证：开发版插件无安装态也能被读取

- [x] 2.2 实现同 `plugin_id` 的“开发版 / 已安装版并存可见、互斥启用”
  - 这一步到底做什么：把第三方同名冲突策略写死到注册表和启停逻辑里
  - 做完以后能看到什么结果：同名第三方插件会同时出现开发版和已安装版两个变体，但同一时刻只允许启用一个
  - 这一步依赖什么：2.1
  - 主要改哪些文件：`app/modules/plugin/service.py`、相关启停服务、相关测试
  - 这一步明确不做什么：不让 `plugins-dev` 获得覆盖 `builtin` 的特权
  - 怎么验证：同名 third_party 变体都能显示；默认开发版启用；切换启用时另一个自动停用

- [x] 2.3 保留 marketplace/local 安装语义
  - 这一步到底做什么：即使同名开发版存在，也保留已安装变体的安装状态和版本治理信息
  - 做完以后能看到什么结果：开发版存在时，已安装版仍然能显示自己是本地安装还是市场安装，以及对应版本状态
  - 这一步依赖什么：2.2
  - 主要改哪些文件：插件服务、市场服务、相关测试
  - 这一步明确不做什么：不让开发版直接改数据库里的安装实例状态
  - 怎么验证：marketplace/local 同名场景下，安装语义仍然完整可见

- [x] 2.4 给开发版生效状态和变体身份增加前端标识
  - 这一步到底做什么：让插件列表、表格和详情都能明确区分“开发版”“已安装版”，并在开发版实际生效时显示“开发插件生效中”
  - 做完以后能看到什么结果：用户能一眼看出当前跑的是开发版还是已安装版，不再靠猜
  - 这一步依赖什么：2.2、2.3
  - 主要改哪些文件：插件注册表 schema、插件服务、前端插件页、前端插件详情
  - 这一步明确不做什么：不新增新的插件来源类型
  - 怎么验证：同名 third_party 和 dev-only 插件场景都能显示正确标签

- [x] 2.5 支持直接切换到已安装版做回归验证
  - 这一步到底做什么：把“验证正式安装版”从手工挪目录改成页面切换启用
  - 做完以后能看到什么结果：开发者不需要再移动 `plugins-dev` 源码，就能切换到已安装版测试
  - 这一步依赖什么：2.2
  - 主要改哪些文件：插件启停逻辑、插件管理页文案、相关文档
  - 这一步明确不做什么：不通过手工重命名目录实现切换
  - 怎么验证：同名开发版和安装版存在时，可直接在页面里来回切换启用

## 阶段 3：守住边界，不把 plugins-dev 变成安装目录

- [x] 3.1 确认启动同步只恢复安装态目录
  - 这一步到底做什么：让启动同步继续只看 `third_party/local` 和 `third_party/marketplace`

  - 这一步依赖什么：2.1
  - 主要改哪些文件：启动同步服务、相关测试
  - 这一步明确不做什么：不让 `plugins-dev` 参与落库

- [x] 3.2 删除安装态时不删除开发源码
  - 这一步到底做什么：保证卸载和清理只作用于安装产物
  - 做完以后能看到什么结果：删除安装态后，`plugins-dev` 同名开发版仍然可见
  - 这一步依赖什么：2.2
  - 主要改哪些文件：插件挂载/清理服务、相关测试
  - 这一步明确不做什么：不为开发目录做额外垃圾清理
  - 怎么验证：删除安装态后开发版仍可见的测试通过

- [x] 3.3 修复测试环境的 `plugin_dev_root` 隔离
  - 这一步到底做什么：让测试使用独立空目录，而不是共享真实仓库 `plugins-dev`
  - 做完以后能看到什么结果：真实仓库 `plugins-dev` 不再污染单测
  - 这一步依赖什么：2.1

  - 这一步明确不做什么：不改业务逻辑
  - 怎么验证：相关插件测试回归通过
