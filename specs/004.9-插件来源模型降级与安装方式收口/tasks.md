# 004.9 任务清单：插件来源模型降级与安装方式收口

这份任务清单只做一件事：把旧 `official` 模型彻底收口到 `builtin` / `third_party + install_method`，并把 `plugins-dev` 变成开发覆盖源。

---

## 阶段 1：收口正式模型

- [x] 1.1 删除 `official` 作为正式运行时类型
  - 这一步到底做什么：统一后端模型、DTO、枚举和兼容入口，只保留 `builtin`、`third_party`
  - 做完以后能看到什么结果：代码里不再把 `official` 当成当前类型继续分支
  - 这一步依赖什么：无
  - 主要改哪些文件：插件模型、schema、服务层、迁移脚本
  - 这一步明确不做什么：不改开发覆盖源行为
  - 怎么验证：搜索正式写路径；相关单测通过

- [x] 1.2 把第三方安装方式收口为 `local` / `marketplace`
  - 这一步到底做什么：把原来的 `manual`、`trusted_level` 等旧口径收口到安装方式
  - 做完以后能看到什么结果：目录生成、安装、升级、卸载都按安装方式走
  - 这一步依赖什么：1.1
  - 主要改哪些文件：安装服务、市场服务、启动同步、清理逻辑
  - 这一步明确不做什么：不新增新的来源类型
  - 怎么验证：本地安装和市场安装相关测试通过

---

## 阶段 2：引入 plugins-dev 开发覆盖源

- [x] 2.1 让 `plugins-dev` 参与家庭插件注册表合并
  - 这一步到底做什么：扫描 `apps/api-server/plugins-dev/`，生成开发态第三方插件注册项
  - 做完以后能看到什么结果：开发中的插件不安装也能出现在家庭插件注册表
  - 这一步依赖什么：1.1、1.2
  - 主要改哪些文件：`app/modules/plugin/service.py`
  - 这一步明确不做什么：不写数据库 mount
  - 怎么验证：开发态插件无 mount 也能被读到

- [x] 2.2 实现同名优先级 `builtin > plugins-dev > installed third_party`
  - 这一步到底做什么：把冲突规则写死到注册表合并逻辑里
  - 做完以后能看到什么结果：开发版能覆盖第三方，但不会覆盖内置插件
  - 这一步依赖什么：2.1
  - 主要改哪些文件：`app/modules/plugin/service.py`、相关测试
  - 这一步明确不做什么：不给 `plugins-dev` 内置插件特权
  - 怎么验证：覆盖 third_party 的测试通过；覆盖 builtin 的测试被拒绝

- [x] 2.3 保留 marketplace/local 安装语义
  - 这一步到底做什么：当 marketplace/local 插件被开发版覆盖时，保留安装状态和版本治理字段
  - 做完以后能看到什么结果：开发覆盖不抹掉安装态信息
  - 这一步依赖什么：2.2
  - 主要改哪些文件：插件服务、市场服务、相关测试
  - 这一步明确不做什么：不让开发覆盖直接改数据库实例状态
  - 怎么验证：marketplace 覆盖测试通过

## 阶段 3：守住边界，不把 plugins-dev 变成安装目录

- [x] 3.1 确认启动同步只恢复安装态目录
  - 这一步到底做什么：让启动同步继续只看 `third_party/local` 和 `third_party/marketplace`
  - 做完以后能看到什么结果：不会为开发目录创建 mount
  - 这一步依赖什么：2.1
  - 主要改哪些文件：启动同步服务、相关测试
  - 这一步明确不做什么：不让 `plugins-dev` 参与落库
  - 怎么验证：startup sync 测试通过

- [x] 3.2 删除安装态时不删除开发源码
  - 这一步到底做什么：保证卸载和清理只作用于安装产物
  - 做完以后能看到什么结果：删安装态后 `plugins-dev` 同名插件仍然可见
  - 这一步依赖什么：2.2
  - 主要改哪些文件：插件挂载/清理服务、相关测试
  - 这一步明确不做什么：不为开发目录做额外垃圾清理
  - 怎么验证：删除安装态后开发版仍可见的测试通过

- [x] 3.3 修复测试环境的 `plugin_dev_root` 隔离
  - 这一步到底做什么：让测试用独立空目录，而不是共享固定夹具目录
  - 做完以后能看到什么结果：真实仓库 `plugins-dev` 不再污染单测
  - 这一步依赖什么：2.1
  - 主要改哪些文件：`tests/test_db_support.py`
  - 这一步明确不做什么：不改业务逻辑
  - 怎么验证：`tests.test_plugin_marketplace_service` 等回归通过
