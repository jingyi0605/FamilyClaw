# 20260310-用户前端实施节奏与里程碑-v0.1

## 1. 这份文档解决什么问题

这份文档只回答两件事：

1. 先做哪一批
2. 做到什么程度算站稳

## 2. 总体节奏

建议按四段推进，不要一上来同时铺十几个页面。

### 阶段 A：搭骨架

目标：

- 建 `apps/user-web`
- 建主路由
- 建主题 token
- 建 i18n
- 建家庭上下文切换

做到这里，说明前端底座站住了。

### 阶段 B：做首页和家庭配置

目标：

- 首页 / 仪表盘
- 家庭概览
- 房间
- 成员
- 设置里的设备与 HA
- 设置里的 AI 配置

做到这里，说明用户前端已经能承担“看状态 + 改配置”的主任务。

### 阶段 C：做助手和记忆

目标：

- 助手多会话
- 连续对话
- 快捷动作
- 记忆列表
- 记忆详情
- 记忆纠错

做到这里，说明这套前端已经有自己的产品中心。

### 阶段 D：做产品化收口

目标：

- 空态、错误态、骨架屏
- 响应式
- 长辈友好模式
- 联调文档和验收记录

做到这里，说明这东西已经不是原型，而是可持续迭代的产品前端。

## 3. 每阶段的最低验收线

### 阶段 A 最低验收线

- `user-web` 能独立构建
- 五个一级入口可导航
- 主题和语言可切换

### 阶段 B 最低验收线

- 首页能稳定展示家庭摘要
- 家庭、房间、成员配置可保存
- 设置里的 HA 和 AI 配置页能工作

### 阶段 C 最低验收线

- 助手支持新建会话和连续追问
- 助手失败时能降级
- 记忆支持查看和纠错

### 阶段 D 最低验收线

- 关键页面有统一空态和错误态
- 长辈模式可用
- 有联调和验收文档

## 4. 当前最值得先做的页面

如果资源有限，优先顺序建议是：

1. 首页
2. 家庭概览
3. 成员
4. 助手
5. 设置里的 AI 配置
6. 记忆

原因很直接：

- 首页建立第一印象
- 家庭和成员页建立主数据维护能力
- 助手建立产品核心价值
- 设置里的 AI 配置决定用户是否真正敢用助手
- 记忆决定系统是否能长期可信

## 5. 当前不建议过早投入的方向

- 高保真视觉稿先做到能定方向就行，不要一开始卷像素
- 不要先做复杂拖拽编排
- 不要先做底层 provider 调试页
- 不要先搞大而全移动端适配

## 6. 实施提醒

- 前端工程一开始就要和 `admin-web` 分家
- 配置页必须完整，但用词必须是用户语言
- 每完成一个阶段，都要补联调说明和阶段验收记录

一句话：

先把骨架做对，再把主任务做稳，最后再追求精致。
