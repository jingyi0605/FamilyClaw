# Spec 002.2 - 家庭用户前端产品化

状态：Draft

## 目标摘要

现在仓库里的 `admin-web` 已经能做很多事，但它的本质是调试台：适合开发、联调、验收，不适合普通家庭成员日常使用。

本 Spec 002.2 的目标，不是把 `admin-web` 换个皮，而是单独规划一套真正面向用户的产品前端：

1. **用户前端独立成应用**：单独建设 `user-web`，和 `admin-web` 明确分家。
2. **首页像家庭工作台，不像运维控制台**：让用户一眼看到家庭、房间、成员、设备和最近事件。
3. **完整配置页面，但必须易用**：家庭入口负责家庭、房间、成员、关系；设置入口负责 HA 对接、AI 配置、主题和语言，但表达方式必须是用户语言，不是技术黑话。
4. **AI 助手成为产品中心能力**：支持多会话、连续对话、快捷动作和可解释降级。
5. **家庭记忆从一开始就有管理入口**：能查看、纠错、标记失效、控制可见范围。
6. **主题与国际化不是后补丁**：从第一阶段就纳入设计 token 和 i18n 骨架。

## 与其他 Spec 的关系

- **依赖 `Spec 001`**：家庭、成员、关系、偏好、权限等主数据。
- **依赖 `Spec 002`**：房间、设备、在家状态、上下文概览、HA 基础对接。
- **协同 `Spec 002.1`**：用户前端会消费问答、提醒、场景和 AI 网关能力，但不承担开发调试职责。
- **承接 `Spec 003`**：家庭记忆页面先落管理入口和展示结构，后续和完整记忆中心能力继续对齐。
- **承接 `Spec 004`**：AI 助手中的提醒、广播、问答入口会优先按 `Spec 004` 的服务语义收敛。
- **预留 `Spec 008`**：长辈友好主题、隐私策略、日报周报等成熟化能力在这里预留前端骨架。

## 当前阶段策略

本 Spec 采用下面这套很明确的策略：

- **调试台和用户端彻底分家**：`admin-web` 继续为开发者服务，`user-web` 只为产品用户服务。
- **先按用户任务组织页面**：首页、家庭、助手、记忆、设置，而不是把后台模块直接平铺。
- **完整配置保留，但技术细节收口**：普通用户能改家庭配置，不需要理解 provider 路由、原始审计日志和底层错误码。
- **主题与国际化先搭骨架**：不接受“先写死中文，后面再国际化”的烂路子。
- **先把高频闭环做稳**：首页、家庭配置、AI 助手、家庭记忆先跑通，再做花哨皮相。

这不是保守，这是好品味。前端产品首先要让人愿意打开，而不是让人看见就想关。
