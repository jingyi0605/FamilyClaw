# Spec 001 - 任务列表

## 任务拆分原则

- 先搭工程骨架，再落数据模型
- 先做后端闭环，再补管理台页面
- 先做可复用底座，再做业务能力

---

## 阶段 A：工程初始化

### T001 初始化后端工程

- 创建 `apps/api-server`
- 创建基础启动入口
- 接入配置管理
- 接入日志系统

完成定义：

- 服务可启动
- 健康检查接口可访问

### T002 初始化数据库与迁移

- 接入 `SQLite`
- 建立迁移工具
- 创建首个迁移目录

完成定义：

- 能执行迁移
- 能回滚迁移

### T003 初始化管理台工程

- 创建 `apps/admin-web`
- 建立基础布局
- 建立路由骨架

完成定义：

- 页面可运行
- 能访问空白管理台

---

## 阶段 B：主数据模型

### T004 创建 `households` 表

- 定义字段
- 创建迁移
- 建立 ORM 模型

完成定义：

- 可插入和查询家庭

### T005 创建 `members` 表

- 定义字段
- 建立与家庭关系
- 建立状态字段

完成定义：

- 可插入和查询成员

### T006 创建 `member_relationships` 表

- 定义关系类型
- 建立唯一约束

完成定义：

- 可新增与查询关系

### T007 创建 `member_preferences` 表

- 定义偏好结构

完成定义：

- 可读写成员偏好

### T008 创建 `member_permissions` 表

- 定义资源类型
- 定义动作类型

完成定义：

- 可读写权限规则

### T009 创建 `rooms`、`devices`、`device_bindings` 表

- 定义房间主数据
- 定义设备主数据
- 定义外部实体绑定

完成定义：

- 可维护房间和设备

### T010 创建 `audit_logs` 表

- 定义审计结构

完成定义：

- 可记录审计日志

### T013 实现关系 API

### T014 实现偏好与权限 API

- 查询偏好和权限

完成定义：

- 偏好和权限均可配置

### T015 实现房间与设备 API

- 创建房间
- 查询房间

完成定义：

- 房间和设备管理可跑通

### T016 接入审计中间件或服务

完成定义：

## 推荐执行顺序

强烈建议按如下顺序执行：

1. `T001` → `T002` → `T003`
2. `T004` → `T010`
3. `T011` → `T016`
4. `T017` → `T019`
5. `T020` → `T023`
6. `T024` → `T027`

---

## 下一步最该做什么

如果你现在准备开始写代码，建议立刻做下面 3 件事：

1. 完成 `T001`、`T002`、`T004`、`T005`
2. 先把 `households` 和 `members` 的 CRUD 跑通
3. 再去对接 `Home Assistant`

也就是说，**第一周最重要的目标不是“让系统聪明”，而是“让系统有家、有成员、有设备”**。
