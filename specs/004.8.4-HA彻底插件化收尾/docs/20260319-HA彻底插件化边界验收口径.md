# 20260319-HA彻底插件化边界验收口径

## 这份文档是干什么的

这份文档只做一件事：

- 把“什么算 HA 彻底插件化完成”写成硬门槛

以后谁再说“差不多了”“主链已经插件化了”“剩下只是清理”，都先拿这份文档对。

## 一、主口径

这次收尾的文档主从关系是：

1. `004.8.4`
   - 收尾总控 spec
2. `005.1`
   - 原则 spec
3. `005.4`
   - 功能清理主战场
4. `005.6`
   - 前端局部收口 spec，不得作为“彻底插件化完成”的证明

## 二、黑名单

下面这些范围默认禁止保留 HA 专名业务逻辑。只要命中，就必须解释。

### 2.1 核心层黑名单

- `apps/api-server/app/modules/**`
- `apps/api-server/app/api/**`

禁止内容：

- `Home Assistant` 专名业务文案
- `homeassistant`、`home_assistant` 平台特判
- 核心直接 import HA 插件 runtime、client 或集成实现
- 用平台名决定控制、同步、状态读取主链

### 2.2 插件层黑名单

- `apps/api-server/app/plugins/builtin/homeassistant_*/**`

禁止内容：

- 直接 import `app.modules.*.repository`
- 直接 import `app.modules.plugin.config_crypto`
- 直接依赖 `_system_context.database_url`
- 在插件内部自建数据库会话读核心表

### 2.3 前端层黑名单

- `apps/user-app/src/**`

禁止内容：

- 顶层 Home Assistant 专名文案
- `home_assistant_status` 这类平台私有字段
- 按 `plugin_id == "homeassistant"` 猜图标、猜流程、猜页面结构
- 旧 HA 专用页面和旧 HA 专用接口语义

## 三、白名单规则

只有下面这些范围可以短期保留 HA 专名，但也必须解释原因和删除条件：

- `apps/api-server/app/plugins/builtin/homeassistant_*`
- 迁移说明、历史 spec、验收文档
- 明确标注为临时桥接的兼容层

白名单要求：

1. 必须精确到目录、文件或调用点
2. 必须写保留原因
3. 必须写删除条件
4. 不能写成“以后再清”“兼容需要”这种废话

## 四、固定检查项

### 4.1 关键词检查

至少检查这些关键词：

- `homeassistant`
- `home_assistant`
- `Home Assistant`
- `HouseholdHaConfig`
- `ha-config`
- `sync/ha`

### 4.2 依赖检查

至少检查这些越界依赖：

- `app.modules.*.repository`
- `config_crypto`
- `database_url`
- 核心直接 import `app.plugins.builtin.homeassistant_*`

### 4.3 状态检查

至少检查这些文档状态：

- `005.4/tasks.md` 关键任务是否仍为 `IN_PROGRESS` 或 `TODO`
- `005.6` 是否被误用为“彻底插件化完成”的证明
- `004.8.4` 的任务状态是否和代码现状一致

## 五、当前已知残留基线

下面这些不是猜测，是当前已经确认的高优先级残留。

- `apps/api-server/app/modules/device/service.py`

  - 仍直接 import `get_home_assistant_runtime_config`
  - 仍按 `instance.plugin_id != "homeassistant"`、`platform == "home_assistant"` 特判
- `apps/api-server/app/modules/device_control/router.py`
  - 已发现 HA 专名常量和专用函数残留
- `apps/api-server/app/modules/integration/service.py`

### 5.2 插件越界

### 5.3 前端越界

- `apps/user-app/src/runtime/h5-shell/i18n/I18nProvider.tsx`

## 六、最终通过条件

只有同时满足下面这些条件，才允许宣布“HA 彻底插件化完成”：

1. `005.4` 关键任务已回写完成，不再存在关键 `IN_PROGRESS`
2. 核心层 HA 专名业务逻辑已清零，或只剩明确白名单
3. 插件层不再直接依赖核心仓储、密钥和数据库内部实现
4. 前端顶层不再保留 HA 专名特例
5. 最终交付附带 grep 结果、白名单清单、测试结果和剩余残留说明

## 七、不允许再说的话

- “主链已经差不多插件化了”
- “剩下只是清理”
- “功能能跑就先算完成”
- “这个先不写进 spec，后面再补”

这些话的共同问题是：它们都在替未完成项遮羞。
