# 20260316-user-app现状样式审计清单

## 这份清单现在记录什么

这不是全仓最终审计结果，而是 `Spec 012` 启动阶段已经确认的几类高价值问题。先把最容易继续扩散、最影响判断的点记下来，后面迁移时才不会重新踩坑。

## 已确认问题

### 1. `designWidth` 已设为 `750`

- 位置：`apps/user-app/config/index.ts:38`
- 现状：全局设计稿基准为 `750`
- 判断：当前没有证据说明这里本身配错，暂时不能把它当成问题源

### 2. H5 配置没有额外 `pxtransform` 覆盖

- 位置：`apps/user-app/config/platform/h5.ts`
- 现状：没有看到 `baseFontSize`、`maxRootSize`、`minRootSize`、`targetUnit`
- 判断：当前 H5 尺寸换算主要还是 Taro 默认行为

### 3. 仓库里已经明确记录过 `rem` 放大问题

- `apps/user-app/src/pages/assistant/index.h5.scss:28`
- `apps/user-app/src/runtime/h5-shell/theme/tokens.ts:73`
- `apps/user-app/src/runtime/h5-shell/styles/index.h5.scss:5999`

判断：

- 这不是新发现的问题
- 说明团队之前已经知道 Taro H5 根字号会把某些手写 `rem` 心智放大

### 4. JSX/TS 行内固定像素还比较多

首批已确认位置：

- `apps/user-app/src/components/AppUi.tsx`
- `apps/user-app/src/components/AuthShellPage.tsx`
- `apps/user-app/src/components/MainShellPage.tsx`
- `apps/user-app/src/pages/login/index.tsx`
- `apps/user-app/src/pages/setup/index.tsx`
- `apps/user-app/src/runtime/guard.tsx`

判断：

- 这些地方的 `'24px'`、`'26px'`、`'40px'` 不是设计稿源码单位
- 它们是浏览器固定像素或运行时样式值
- 需要优先纳入共享组件或 token 收口范围

### 5. 手写 `rem` 仍然存在于 H5 页面和壳层样式

首批已确认位置：

- `apps/user-app/src/pages/home/index.h5.scss`
- `apps/user-app/src/pages/setup/WelcomeStep.css`
- `apps/user-app/src/runtime/h5-shell/styles/index.h5.scss`

判断：

- 这些地方需要分两类处理
- 一类是历史包袱，短期先留兼容说明
- 一类是新写法，需要回收成 `px` 设计稿单位或 token 驱动尺寸

## 当前最应该优先处理的对象

### P1：`AppUi.tsx`

原因：

- 它是公共组件入口
- 里面已经有多处固定字号和间距
- 不先收口这里，后面页面迁移会继续复制旧写法

### P1：壳层主题与壳层样式

原因：

- `tokens.ts` 和 `index.h5.scss` 已经承担了全局语义
- 如果共享层不上移，这里会长期和 `packages/user-ui` 并行长两套标准

### P1：`assistant` / `setup-guard` 这类已经出现尺寸异常的区域

原因：

- 这些位置已经有明确的“避坑注释”
- 适合反向提炼正式规则，而不是继续靠局部补丁

## 当前不应该急着做的事

### 1. 不要先全局改 `designWidth`

原因：

- 还没有证据表明设计稿基准本身配错
- 现有问题更像是单位语义混用

### 2. 不要先把所有页面一起重写

原因：

- 真正的瓶颈是标准源没收口
- 标准层没立住，页面改得越多，返工越多

### 3. 不要继续让行内 `'24px'` 扩散

原因：

- 这是最容易复制、最难统一回收的写法

## 下一步建议

1. 继续补完整的单位审计，至少分清手写 `rem`、样式文件 `px`、行内固定像素三类来源
2. 先把 `packages/user-ui` 里的 token 分层和导出入口补齐
3. 设计 `AppUi.tsx` 到共享层的归并方案
4. 给新增代码补最小可执行检查，先拦住新的手写 `rem` 和新的行内固定尺寸
