## 页面基线检查

### 1. 公共壳层

- `AppShellPage`、`AuthShellPage`、`MainShellPage` 不再直接维护一套散装按钮、卡片、正文样式
- 壳层状态块统一使用共享 `PageSection` / `StatusCard` / `UiText`
- 公共壳层不再出现明显的 `'24px'`、`'40px'`、`'140px'` 这类 JSX 固定尺寸字符串

### 2. 登录入口

- 登录页输入区统一使用 `FormField` + `UiInput`
- 登录提交按钮统一使用 `UiButton`
- 登录错误提示统一使用共享文本风格，不再手写一套原生 `Text`

### 3. 设置页

- 设置页入口壳统一使用 `PageHeader`
- 设置页基础区块统一使用 `PageSection`
- 开关行统一使用共享 `ToggleSwitch`
- AI 设置页空态、提示卡、标签已经接回共享层

### 4. setup / home / assistant 第一轮迁移

- setup 和 family 的公共 `PageHeader / Card / EmptyState / Section / Toggle` 已回收到共享层兼容包装
- home 高复用 `Card / EmptyState` 已接到共享层
- assistant 高复用 `EmptyState` 已接到共享层
- 这些页面仍然保留的重业务块被明确当成历史债，不再当成新的公共标准

## 单位和样式规则检查

- 受保护文件中禁止新增手写 `rem`
- 受保护文件中禁止新增 JSX/TS 固定 `px` 字符串
- 规则已经接入 `apps/user-app` 的 `pretypecheck`
- 已知仍存在 `rem` 热点的旧文件只保留在历史债目录，不在这轮假装清完

## 已知边界

- `settings/channel-access` 和 `settings/components/*` 仍有大量业务专有卡片，后续继续拆
- `setup/index.h5.tsx` 的表单内部样式还没完全吃到共享表单块
- `home` 和 `assistant` 仍保留业务专有内容卡，不属于“共享基础件缺失”
- 这轮没有碰 `LegacyFamilyPage.tsx`

## 收口判断

满足下面四条，就可以认为 spec012 这轮已经收住：

1. 共享层能直接支撑公共壳层、登录页、设置页入口和第一批高频页面迁移
2. 新的页面公共块默认从 `packages/user-ui` 获取，不再回到页面私有 `base.tsx`
3. 防回潮脚本已接入并能实际拦住最常见的坏写法
4. 已迁移区域、未迁移区域和剩余边界都已经写清楚，不再口头传递
