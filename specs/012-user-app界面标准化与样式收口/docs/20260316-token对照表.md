# 20260316-token对照表

这份表不是理论说明，是给后续改页面的人快速查“旧写法现在该落到哪里”。

## 先记住一条

- 现在唯一上游标准源在 `packages/user-ui/src/theme/`
- `apps/user-app/src/runtime/h5-shell/theme/` 只负责 H5 主题状态和 CSS 变量映射，不再定义另一套 canonical token
- 页面层优先吃共享组件；只有共享组件撑不住时，才直接消费 token

## 分层对照

| 旧认知 | 现在对应位置 | 说明 |
| --- | --- | --- |
| `userAppTokens.color* / spacing* / radius* / fontSize*` | `userAppFoundationTokens` + `userAppSemanticTokens` | `userAppTokens` 现在只是兼容入口，老代码还能跑，新代码别继续把它当第一选择 |
| H5 主题值 `apps/user-app/src/runtime/h5-shell/theme/tokens.ts` | `packages/user-ui/src/theme/themes.ts` | H5 私有文件只保留兼容导出和映射职责 |
| 页面里自己写按钮颜色、卡片边框、输入框圆角 | `userAppComponentTokens.button/card/input` | 先改成共享组件；只有组件暂时接不住时才考虑直接读 component token |
| 页面里自己写标题字号、正文颜色、次级文本颜色 | `userAppComponentTokens.text` | 标题、正文、说明文案优先走 `UiText` 变体 |
| 页面里自己写区块标题 + 卡片壳 + 描述文案 | `PageSection` / `PageHeader` | 不再每页自建 `Section`、`PageHeader` |
| 页面里自己写空态卡片 | `EmptyStateCard` | 空态卡片的文案、动作、图标都走共享层 |
| 页面里自己写开关行 | `ToggleSwitch` | 设置页和设备设置统一吃共享开关行 |

| --- | --- |

## 还允许直接碰 token 的场景

- 共享组件已经有了，但页面只是在做布局间距微调
- 业务块视觉很特殊，暂时进不了共享层，但仍要复用统一颜色、字号、圆角
- H5 样式文件里需要消费 CSS 变量，而不是在 JSX 里塞固定尺寸

## 明确不要再做的事

- 不要再在页面目录里补新的 `Card / EmptyState / PageHeader / Toggle` 私有基础块
- 不要再把 `runtime/h5-shell/theme` 当成标准 token 源
- 不要再把 `'24px'`、`'40px'`、`0.75rem` 这种值直接塞进 JSX/TS
- 不要为了修某一页尺寸异常去改全局 `designWidth`
