# 补充文档目录 - 基于 GitHub 仓库的插件市场、一键安装与手动启用

状态：Draft

## 这里后续应该补什么

这次 `docs/` 不该堆空话，只补真正会被反复查的东西。

建议后续按需要补这几类文档：

1. `市场仓库示例结构.md`
   - 官方市场仓库目录示例
   - 单插件目录示例
   - `market.json` 和 `entry.json` 示例

2. `插件源码仓库最小模板.md`
   - `manifest.json`
   - `README.md`
   - `requirements.txt`
   - `plugin/`
   - `tests/`

3. `注册表条目字段说明.md`
   - 每个字段的作用
   - 哪些字段必须和 manifest 对齐
   - 哪些字段只是市场展示字段

4. `市场接入与联调说明.md`
   - 如何添加第三方市场仓库
   - 如何手动触发同步
   - 如何排查仓库结构错误和条目错误

5. `验收清单.md`
   - 官方市场默认可见
   - 第三方市场可添加
   - 同名插件来源不丢失
   - 一键安装成功后默认未启用
   - 手动启用前必须配置完成

## 不要往这里堆什么

- 不要把 `requirements.md` 再抄一遍
- 不要把开发者文档正文复制一遍
- 不要写“以后再补”的空文件

## 一条原则

这里的补充文档只回答一个问题：

实现、联调、提 PR 的人最容易卡在哪。

答不上这个问题的文档，就别建。
