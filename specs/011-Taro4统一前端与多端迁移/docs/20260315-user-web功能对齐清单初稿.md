# 20260315-user-web功能对齐清单初稿

## 这份清单现在解决什么问题

这不是验收成品，而是迁移启动后的第一版对齐底稿。

它先回答三件事：

1. `user-web` 现在到底有哪些主入口
2. 哪些入口已经在 `user-app` 里有壳了
3. 哪些东西现在不能硬搬，必须先继续抽共享层

## 冻结规则

从 `spec011` 开始，`apps/user-web` 进入“只修不增”阶段。

允许做的事：

- 修线上阻断级 bug
- 为新旧入口切换补临时兼容桥接
- 为共享层抽离做必要的小改动

不允许做的事：

- 承接正式新功能
- 继续堆浏览器特有能力当长期方案
- 为了赶进度把整页直接复制到 `apps/user-app`

## 这轮已经抽出的共享层

- `packages/user-core`
  - 启动链路、首页、家庭、设置会用到的核心类型和领域模型
  - 共享 API client：认证、家庭、房间、成员、设备、上下文、提醒、locale
  - 启动摘要加载服务
  - 认证清理、家庭摘要、setup、主题偏好、语言偏好 helper
- `packages/user-platform`
  - 浏览器与 Taro 存储适配
  - 平台目标探测
  - 实时协议、URL 构造和浏览器 websocket client
- `packages/user-ui`
  - 页面分区组件
  - 状态卡片组件
  - 基础视觉 token
- `packages/user-testing`
  - 对齐清单 registry
  - 当前页面迁移状态初稿

## 暂时不要硬搬的东西

下面这些不是不能做，而是现在硬搬会把旧问题一起带过来：

- 助手页里的聊天 UI 和会话编排
- 记忆页里的复杂列表和纠错交互
- 设置页里通知、插件、通讯接入这些低频子页
- 家庭页的大表单和复杂关系编辑 UI

先继续抽共享 API、状态和平台接口，再迁页面壳。顺序不能反。
