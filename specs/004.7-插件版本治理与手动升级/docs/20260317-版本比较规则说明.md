# 版本比较规则说明

## 这次只做最小可用规则

这轮不是做完整 semver 平台。

当前后端统一版本比较规则只支持两类常见写法：

- 纯数字点分版本，比如 `1.0.0`
- 带常见预发布后缀的版本，比如 `1.0.0rc1`、`1.0.0-beta1`

## 比较顺序

1. 先比较主版本号、次版本号、补丁号
2. 正式版高于预发布版
3. 预发布按 `alpha < beta < rc < 正式版`

## 当前保守边界

如果版本字符串超出当前规则，比如项目后面突然塞进一套奇怪命名：

- 不瞎猜
- 不继续给“可升级”结论
- 统一降级成 `unknown`

这比乱比一通然后误判要安全得多。

## 宿主兼容判断

宿主兼容只看：

- 当前宿主版本 `settings.app_version`
- 市场版本条目里的 `min_app_version`

规则很直接：

- `host_version >= min_app_version`：兼容
- `host_version < min_app_version`：阻断
- `min_app_version` 缺失或当前规则无法解析：按保守策略处理，返回 `unknown`，安装和版本切换直接拦住
